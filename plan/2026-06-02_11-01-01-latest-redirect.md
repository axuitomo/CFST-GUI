# Plan: manifest download_url 改用 latest 静态重定向

## 背景

当前 `release_asset_download_url()` 生成的是版本绑定的 URL：
`https://github.com/axuitomo/CFST-GUI/releases/download/v1.7.4/cfst-gui-windows-amd64.exe`

GitHub 提供 `/releases/latest/download/{asset}` 静态 302 重定向，永远指向最新 release 的资产。改用此模式后 manifest 中的 download_url 不再随版本变化，客户端也能用稳定 URL 下载。

## 改动范围

### 1. `scripts/build-release.sh` — `release_asset_download_url()`

**改前**（第 172–176 行）：
```bash
release_asset_download_url() {
  local asset_name="$1"
  local repository="${GITHUB_REPOSITORY:-axuitomo/CFST-GUI}"
  local release_tag="v${VERSION#v}"
  printf 'https://github.com/%s/releases/download/%s/%s' "$repository" "$release_tag" "$asset_name"
}
```

**改后**：
```bash
release_asset_download_url() {
  local asset_name="$1"
  local repository="${GITHUB_REPOSITORY:-axuitomo/CFST-GUI}"
  printf 'https://github.com/%s/releases/latest/download/%s' "$repository" "$asset_name"
}
```

移除 `release_tag` 变量，URL 路径从 `/releases/download/{tag}/` 改为 `/releases/latest/download/`。

### 2. `internal/app/update.go` — 客户端 fallback

在 `selectReleaseAsset()` 的 fallback 分支（第 185–193 行）中，当 manifest 不存在且按命名规则也找不到 asset 时，构造 latest URL 作为兜底：

```go
name := defaultReleaseAssetName(runtime.GOOS, runtime.GOARCH)
asset, ok := assetMap[name]
if !ok || asset.BrowserDownloadURL == "" {
    // 构造 latest 静态重定向 URL 作为兜底
    return updateManifestAsset{
        DownloadURL: latestAssetDownloadURL(name),
        GoArch:      runtime.GOARCH,
        GoOS:        runtime.GOOS,
        InstallMode: defaultInstallMode(runtime.GOOS),
        Name:        name,
        Platform:    runtime.GOOS + "/" + runtime.GOARCH,
    }, nil
}
```

新增 helper：
```go
const latestReleaseBase = "https://github.com/axuitomo/CFST-GUI/releases/latest/download/"

func latestAssetDownloadURL(assetName string) string {
    return latestReleaseBase + url.PathEscape(assetName)
}
```

同时将 `latestAssetDownloadURL` 加入 `githubDownloadCandidates()` 的候选列表，让代理回退链也覆盖 latest URL。

### 3. `internal/app/update_test.go` — 补充测试

- 测试 `latestAssetDownloadURL()` 输出格式
- 测试 `selectReleaseAsset()` 在无 manifest 且 API 无 BrowserDownloadURL 时能用 latest URL 兜底

## 不改动的部分

- `CheckForUpdates()` 仍通过 API 比较版本号（latest URL 不提供版本信息）
- `downloadAndInstallUpdate()` 的下载逻辑不变（仍走 `downloadFile` + `githubDownloadCandidates`）
- SHA256 校验逻辑不变（manifest 仍包含 sha256 字段）

## 风险

- GitHub `/releases/latest/download/` 返回 302，`http.Client` 默认跟随重定向，无额外处理
- 若资产名含特殊字符，`url.PathEscape` 会编码，但当前所有资产名均为 ASCII 安全字符
- latest URL 指向"最新 release"，如果用户本地版本高于最新 release（预发布场景），API 已通过版本比较拦截，不会误触发下载
