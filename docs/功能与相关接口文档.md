# CFST-GUI 功能与相关接口文档

## 1. 文档说明

本文档基于当前仓库源码整理 CFST-GUI 的功能、运行链路和接口边界。当前项目是一个基于 Wails v2.12.0 + Vue 的桌面端 Cloudflare/CDN IP 测速工具，后端 Go 目标版本为 1.26.2。

当前接口边界如下：

- 桌面 GUI 默认通过 Wails 启动，前端优先经 `window.go.app.App` 调用 Go 后端方法，并兼容旧生成物的 `window.go.main.App`。
- Linux WebUI 通过 `webui` build tag 提供本地 HTTP Server、REST API、SSE 进度事件和浏览器页面路由；发行时提供 `linux/amd64` 与 `linux/arm64` 两种 bundle，并同时支持 Docker Compose 与 `run-local.sh` 本地运行。
- 文档中的“接口”主要指 Wails 后端绑定方法、WebUI HTTP API、前端 bridge 适配函数、Wails/SSE 事件、桌面配置结构以及外部 HTTP 请求。

关键入口文件：

| 文件 | 作用 |
| --- | --- |
| `main.go` | 根目录薄入口，注入嵌入资源并调用 `internal/app.Run` |
| `resources.go`、`frontend_assets.go`、`tray_icon*.go` | 根目录资源桥接，保持 `frontend/dist` 和 `build/` 图标嵌入路径稳定 |
| `internal/app/run.go` | GUI/CLI 启动判定、CLI 分发和版本信息 |
| `internal/app/gui.go` | Wails 桌面窗口启动与 `App` 绑定 |
| `internal/app/webui.go` | Linux WebUI HTTP 服务、API 分发、SSE 和受限文件访问 |
| `internal/app/app.go` | Wails 后端桥接、配置读写、测速任务执行 |
| `internal/app/app_archive.go` | 配置压缩包、本地备份、WebDAV 备份与还原 |
| `internal/app/storage.go` | 储存目录、输入源档案和策略档案持久化 |
| `internal/app/desktop_colo_dictionary.go` | 桌面端 COLO 字典状态、拉取和本地处理 |
| `internal/app/desktop_sources.go` | 桌面输入源读取、预览、抓取和 MICS抽样处理 |
| `internal/app/desktop_probe_events.go` | `desktop:probe` 事件封装与推送 |
| `frontend/src/lib/bridge.ts` | 前端到 Wails 后端的桥接适配层 |
| `frontend/src/App.vue` | 桌面 UI 状态编排和页面调用入口 |
| `mobileapi/` | Android gomobile 服务实现，与桌面端保持同构接口 |
| `mobile/android/app/src/main/java/io/github/axuitomo/cfstgui/CfstPlugin.kt` | Capacitor `Cfst` plugin，转发前端调用到 gomobile |
| `internal/task/` | CFST TCP、追踪、文件测速核心逻辑 |
| `internal/utils/` | CSV 导出、调试日志、进度工具 |

## 2. 功能总览

### 2.1 桌面 GUI

无参数运行时，程序启动 Wails 桌面 GUI。前端由 Vue 3 + Vite + Tailwind CSS 构建，后端由 Go 绑定 `App` 实例给 Wails。

桌面端主要页面如下：

| 页面 | 功能 |
| --- | --- |
| 任务看板 | 启动探测任务、展示进度、事件过程、警告和导出历史 |
| 当前结果 | 展示当前测速任务的结果表格、移动端结果卡片、排序过滤和导出位置 |
| 输入源 | 管理 URL、本地文件、手动输入三类 IP 来源 |
| 系统配置 | 保存 Cloudflare、导出、探测策略、高级 HTTP 和调试参数 |
| DNS 读取 | 通过 Cloudflare 官方 API 读取当前 Zone、当前配置记录或指定子域名记录；页面本身不执行推送 |

### 2.2 桌面测速

桌面测速复用 CFST 核心流程，并通过 Wails 事件把任务进展同步给前端。

| 功能 | 当前行为 | 关键代码 |
| --- | --- | --- |
| 阶段 0 IP池 | 读取输入源、清洗并展开候选 IP/CIDR/域名，并推送 `probe.preprocessed` | `internal/app/desktop_sources.go`、`internal/app/app.go` |
| 阶段 1 TCP测延迟 | 强制执行 TCP 建连测速，`delayMs` 定义为 TCP 平均延迟 | `internal/task/tcping.go`、`internal/app/app.go` |
| 阶段 2 追踪探测 | 对 TCP 通过候选执行 GET `/cdn-cgi/trace`，GUI 默认并发 30，最大并发 30，用于筛选和地区码 | `internal/task/head.go`、`internal/task/colo.go` |
| 阶段 3 文件测速 | `full` 策略追加文件下载测速，可按配置使用受限并发；下载时间按单 IP 生效 | `internal/task/download.go`、`internal/app/app.go` |
| 阈值过滤 | 支持 TCP 延迟上限/下限、丢包率上限（硬上限 100%，默认 15%）、下载速度下限 | `internal/utils/csv.go`、`internal/app/app.go` |
| 地区码识别 | 追踪响应 body `colo=XXX` 优先，`CF-RAY` 正则备用，并兼容 CloudFront、Fastly、Gcore、CDN77、Bunny 响应头 | `internal/task/colo.go` |
| CSV 导出 | 默认导出 `result.csv`，可配置目标目录和文件名 | `internal/utils/csv.go`、`internal/app/app.go` |
| 调试抓包 | 支持自定义 User-Agent、Host Header、SNI，并可单独启用/关闭调试拨号目标 | `internal/httpcfg/profile.go`、`internal/task/request_profile.go` |

### 2.3 输入源管理

输入源随桌面配置保存，每个来源可独立设置启用状态、类型、IP 上限和 IP 处理模式。内容会按行清洗，跳过空行和 `#` 注释；行内 `#` 后的备注会被丢弃，再从主体内容中提取 IP/CIDR 或域名。域名使用系统本地 DNS 解析为 A/AAAA 后参与测速。

| 输入源类型 | 字段 | 读取方式 |
| --- | --- | --- |
| `url` | `url` | 使用 HTTP GET 读取远程 IP 列表；可省略协议，默认按 `https://` 处理 |
| `file` | `path` | 使用本地文件路径读取 IP 列表 |
| `inline` | `content` | 使用文本框中的手动输入内容 |

候选 IP 处理模式：

| 模式 | 行为 |
| --- | --- |
| `traverse` | 解析 IP/CIDR/域名，按顺序展开 CIDR，并按 `ip_limit` 截断 |
| `mcis` | 界面显示为 MICS抽样，将 IP/CIDR/域名解析后的候选转换为 CIDR 输入，先通过内置抽样搜索引擎和可选 IATA COLO 白名单筛选候选，再交给 CFST 做最终测速 |

### 2.4 探测策略

桌面端配置中有两个主要策略：

| 策略 | 说明 |
| --- | --- |
| `fast` | 默认策略，执行阶段 0/1/2：IP池、TCP测延迟、追踪探测，跳过文件测速 |
| `full` | 执行阶段 0/1/2/3：在追踪通过后追加文件测速 |

阶段 1 TCP 默认发包 4 次，启用跳过首包时首包不计入平均延迟和丢包统计。当前结果页、前端 `tcp_latency_ms` 和 CSV 延迟列只展示 TCP 平均延迟；阶段 2 追踪延迟通过 `traceDelayMs` / `trace_latency_ms` 提供给 UI，CSV 保持旧列格式。

阶段 2 追踪探测使用独立并发闸门，默认并发为 30，最大并发为 30。旧配置中 `concurrency.stage2` 超过 30 时会归一化为 30。阶段 3 文件测速只在 `full` 策略运行，外层文件测速并发固定为 1；单 IP 内部默认使用 4 个 HTTP Range GET 分片聚合测速，服务端不支持 Range 时回退完整流式 GET。文件测速在单 IP 时长内遇到 EOF、短文件完成或临时断流时会自动对同一 IP 续连，并累计预热后的有效测量窗口。

高级参数包括 TCP并发线程、测速上限、单 IP 下载测速时间、下载预热时间、GET 分片并发、下载协议、下载缓冲、端口、文件测速URL、追踪 URL、User-Agent、Host Header、SNI、通用请求 Headers、追踪有效状态码、地区码过滤、调试抓包开关和目标等。旧配置中的 `download_count` 仅作为兼容读取字段；`stage_limits.stage3` 仍作为完整模式进入文件测速的候选上限。

### 2.5 DNS 读取页与推送链路

系统配置中提供独立 Cloudflare 配置卡片，包含启用开关、API Token、Zone ID、记录名、TTL、代理开关、备注、Top N 和分流规则；旧配置中的记录类型字段继续兼容读取。

当前实现行为：

- 桌面和移动端都会通过后端调用 Cloudflare 官方 API，读取记录要求完整 API Token 和 Zone ID。
- `listDnsRecords()` 可读取当前 Zone 全部记录、当前配置记录名，或指定子域名/记录名，并可按 A/AAAA 类型筛选后返回 `DnsRecordSnapshot`。
- DNS 读取页只调用读取接口，不创建、更新或删除记录。
- `pushDnsRecords()` 保留给工作流 `deliver_dns` 节点、定时任务和测速后自动推送；它会将输入 IP 归一化并按地址族分组，IPv4 覆盖同步 A 记录，IPv6 覆盖同步 AAAA 记录。
- TTL 仅支持 60、300、600 秒三档，默认 300 秒；旧配置中的自动 TTL `1` 会归一化为 300 秒。
- 推送操作不是 dry run；启用前应确认记录名、TTL、代理开关、备注、分流规则和 Top N，避免覆盖生产记录。

### 2.6 配置、档案与更新

系统配置页同时承担应用信息、应用数据目录状态、导出目录、备份还原、在线更新和高级探测参数管理。

| 功能 | 当前行为 | 关键代码 |
| --- | --- | --- |
| 应用数据目录 | 默认使用 `os.UserConfigDir()/CFST-GUI`，旧 `storage_dir` 只用于一次性迁移；支持 `CFST_GUI_PORTABLE_ROOT` 便携模式 | `internal/app/storage.go` |
| 输入源档案 | `source-profiles.json` 保存输入源档案，切换档案会写回当前输入源配置 | `internal/app/storage.go`、`internal/app/app.go` |
| 配置导入导出 | JSON 导出包含配置快照、source profiles、storage 状态；ZIP 归档内固定包含 `cfst-gui-config.json` | `internal/app/app.go`、`internal/app/app_archive.go` |
| WebDAV 备份 | 使用 `HEAD` 测试连接、`PUT` 覆盖远端配置包、`GET` 拉取并导入配置包 | `internal/app/app_archive.go`、`mobileapi/archive.go` |
| COLO 字典 | 支持查看字典状态、拉取 geofeed 原始数据并本地处理，供输入源 COLO 过滤和 MICS抽样使用 | `internal/app/desktop_colo_dictionary.go`、`internal/colodict/` |

## 3. 运行链路

### 3.1 GUI 启动链路

```text
main()
  -> app.Run(os.Args[1:], runtimeResources())
  -> runGUI()
  -> wails.Run(...)
  -> Bind: App
  -> frontend 调用 window.go.app.App.*
```

关键点：

- 根目录 `main.go` 是薄入口，资源桥接保留在根目录以维持 `go:embed` 可用路径。
- `internal/app/run.go` 中没有参数时默认进入 `runGUI()`。
- `internal/app/gui.go` 接收根目录注入的 `frontend/dist` 资源，并把 `App` 绑定给 Wails。
- 前端通过 `frontend/src/lib/bridge.ts` 封装 Wails 调用和事件监听，优先使用 `window.go.app.App`，并兼容 `window.go.main.App`。

### 3.2 桌面探测链路

```text
App.vue launchProbe()
  -> bridge.startProbe(payload)
  -> App.RunDesktopProbe(payload)
  -> prepareDesktopSources()
  -> emitter: probe.preprocessed(stage0_pool)
  -> App.runProbe()
  -> stage1_tcp: task.NewPing().Run().FilterDelay().FilterLossRate()，丢包率默认 <=15%，最高可配置到 <=100%
  -> stage2_trace: task.TestTraceAvailability()
  -> stage3_get: task.TestDownloadSpeed(...) 仅 full 策略运行
  -> utils.ExportCsv(...)
  -> emitter: probe.partial_export
  -> emitter: probe.completed
  -> bridge 缓存 taskSnapshots / taskResults
  -> App.vue 刷新看板、当前结果页和自动推送/导出状态
```

### 3.3 输入源预览和抓取链路

```text
App.vue inspectSource()
  -> previewDesktopSource() 或 fetchDesktopSource()
  -> App.PreviewDesktopSource() 或 App.FetchDesktopSource()
  -> processDesktopSource()
  -> loadDesktopSourceContent()
  -> buildDesktopSourceEntriesWithConfig()
  -> 可选 MICS抽样搜索
  -> 返回 preview_entries、source_status、summary
```

远程 URL 输入源会使用系统代理环境变量（如 `HTTP_PROXY`、`HTTPS_PROXY`、`NO_PROXY`），并对网络错误、读取错误、`429` 或 `5xx` 做一次重试。GitHub Raw 地址可在输入源卡片中切换为 jsDelivr CDN；Raw 读取失败时也会自动尝试等价的 jsDelivr 地址。

预览和抓取的主要区别：

| 操作 | 是否持久化状态 | 典型用途 |
| --- | --- | --- |
| 预览 | 默认不持久化，除非 `persist_state` 为真 | 检查来源是否可读、候选是否正确 |
| 抓取 | 持久化 `last_fetched_at`、`last_fetched_count`、`status_text` | 更新输入源读取状态 |

## 4. 后端能力接口

桌面 Wails 后端绑定对象是 `App`，前端可通过 `window.go.app.App` 调用公开方法，并保留 `window.go.main.App` 的兼容判断。WebUI 通过 `POST /api/app/{method}` 分发到同名或等价 `App` 方法。Android 通过 Capacitor `Cfst` plugin 调用 `mobileapi.Service`，对前端保持相同业务能力。

### 4.1 统一返回结构

桌面端和 WebUI 主要使用 `DesktopCommandResult`，Android native 层返回同构 JSON 后由前端 bridge 归一化：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | `string` | 业务状态码 |
| `data` | `any` | 返回数据 |
| `message` | `string` | 用户可读消息 |
| `ok` | `boolean` | 是否成功 |
| `schema_version` | `string` | Go 后端当前为 `cfst-gui-wails-v1` |
| `task_id` | `string \| null` | 关联任务 ID |
| `warnings` | `string[]` | 非致命警告 |

前端 bridge 会把返回结果归一化为自己的 `CommandResult<T>`，前端默认 schema fallback 是 `phase1-bridge-v1`。

### 4.4 `DesktopSourcePreviewPayload`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `config` | `object` | 当前桌面配置快照 |
| `persist_state` | `boolean` | 是否持久化来源读取状态 |
| `preview_limit` | `number` | 返回预览条数上限，默认 16 |
| `source` | `DesktopSource` | 单个输入源配置 |

返回 `data` 结构：

| 字段 | 说明 |
| --- | --- |
| `preview_entries` | 预览候选 IP 列表 |
| `source_status` | 更新后的来源状态 |
| `port_summary` | 端口上下文，包含 `global_tcp_port`、`source_port_values`、`current_test_port`、`port_policy` |
| `summary.action` | `预览` 或 `抓取` |

## 5. WebUI HTTP API

Linux WebUI 通过 `go build -tags webui` 或 `scripts/build-release.sh linux|linux-amd64|linux-arm64` 构建。Release bundle 提供 `linux/amd64` 与 `linux/arm64` 两种产物；Docker Compose 默认监听 `0.0.0.0:34115`，`run-local.sh` 默认监听 `127.0.0.1:34115`。鉴权由 `CFST_WEBUI_TOKEN` 控制，详见 [Docker 与环境变量](./docker-env.md)。

| 路由 | 方法 | 鉴权 | 说明 |
| --- | --- | --- | --- |
| `/api/health` | `GET` | 不需要 | 返回 `ok`、`service`、`version`、`auth_required` |
| `/api/app/{method}` | `POST` | 需要 | 读取 JSON payload，并分发到 `invokeWebUIAppMethod()` 中列出的应用方法 |
| `/api/events/probe` | `GET` | 需要 | SSE 流，逐条发送 `desktop:probe` envelope JSON |
| `/api/files/list?path=...` | `GET` | 需要 | 列出允许根目录内的文件，返回 `entries`、`path`、`roots` |
| `/api/files/download?path=...` | `GET` | 需要 | 下载允许根目录内的单个文件 |
| `/` | `GET`/`HEAD` | 不需要 | 返回嵌入的 `frontend/dist` SPA；找不到静态文件时回退到首页 |

WebUI 鉴权支持 `Authorization: Bearer <token>`，SSE 和下载场景也兼容 `?token=<token>`。如果未设置 `CFST_WEBUI_TOKEN`，受保护 API 会跳过鉴权。

`/api/app/{method}` 当前按能力分发到这些方法组：

| 能力 | WebUI method |
| --- | --- |
| 应用信息与更新 | `GetAppInfo`、`CheckForUpdates`、`DownloadAndInstallUpdate`、`OpenReleasePage` |
| 配置与储存 | `LoadDesktopConfig`、`SaveDesktopConfig`、`LoadDesktopDraft`、`SaveDesktopDraft`、`DiscardDesktopDraft`、`SetStorageDirectory`、`CheckStorageHealth` |
| 配置导出与归档 | `ExportConfig`、`BackupCurrentConfig`、`ExportConfigArchive`、`ImportConfigArchive` |
| WebDAV | `TestWebDAV`、`BackupConfigToWebDAV`、`RestoreConfigFromWebDAV` |
| 输入源档案 | `LoadSourceProfiles`、`SaveSourceProfile`、`UpdateCurrentSourceProfile`、`SaveSourceProfileStore`、`SwitchSourceProfile`、`DeleteSourceProfile` |
| 输入源与 COLO 字典 | `PreviewDesktopSource`、`FetchDesktopSource`、`LoadColoDictionaryStatus`、`UpdateColoDictionary`、`ProcessColoDictionary` |
| 探测任务与结果 | `RunDesktopProbe`、`CancelProbe`、`ResumeProbe`、`ListResultFile` |
| Cloudflare 和 GitHub | `ListCloudflareDNSRecords`、`PushCloudflareDNSRecords`、`TestGitHubExport`、`ExportResultsToGitHub` |
| 调度与路径 | `LoadSchedulerStatus`、`OpenPath` |

未知 method 返回 400 JSON 错误。

文件 API 的允许根目录来自 `/data`、当前 `storageRoot()` 和 `CFST_WEBUI_ALLOWED_ROOTS`。路径会转成绝对路径并校验必须位于允许根内，避免浏览器任意读取宿主文件。

## 6. 前端 Bridge 接口

前端 bridge 文件是 `frontend/src/lib/bridge.ts`。它负责：

- 在 Wails、WebUI、Android native 三种运行时之间选择正确后端。
- 校验并归一化 Wails/WebUI/Capacitor 返回值。
- 将 Go 结构转换为 UI 更容易消费的数据结构。
- 维护当前会话内存态 `taskSnapshots`、`taskResults`。
- 监听 `desktop:probe` Wails 事件、WebUI SSE 或 Capacitor listener 并分发给页面。

```ts
  schema_version: string;

```

## 7. 探测事件接口

后端事件名固定为 `desktop:probe`。桌面端通过 Wails `EventsEmit` 推送，WebUI 通过 `/api/events/probe` SSE 传递，Android 通过 Capacitor `addListener("desktop:probe")` 回传。

### 7.1 事件 Envelope

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `event` | `string` | 事件类型 |
| `payload` | `object` | 事件负载 |
| `schema_version` | `string` | Go 后端 schema |
| `seq` | `number` | 单任务内递增序号 |
| `task_id` | `string` | 任务 ID |
| `ts` | `string` | RFC3339 时间 |

### 7.2 事件类型

| 事件 | payload | 说明 |
| --- | --- | --- |
| `probe.preprocessed` | `accepted`、`filtered`、`invalid`、`source_statuses`、`stage`、`total` | 输入源预处理完成，`stage` 为 `stage0_pool` |
| `probe.progress` | `stage`、`processed`、`passed`、`failed`、`total` | TCP、追踪或文件测速阶段进度 |
| `probe.speed` | `stage`、`ip`、`current_speed_mb_s`、`current_ready`、`average_speed_mb_s`、`average_ready`、`bytes_read`、`elapsed_ms`、`colo` | 阶段 3 文件测速实时速率样本；ready 为 false 时前端显示 `-`，`average_ready=true` 表示已有预热后的有效测量窗口 |
| `stage.detail` | `stage`、`ip`、`reason`、`get.segment_id`、`get.range_start`、`get.range_end`、`get.reconnect_reason` | 调试日志事件；阶段 3 会记录 Range 分片、协议、GET 并发和续连原因 |
| `probe.partial_export` | `target_path`、`written` | CSV 已写出 |
| `probe.completed` | `exported`、`failed`、`passed`、`result_count`、`target_path`、`failure_summary`、`task_context` | 任务完成 |
| `probe.failed` | `message`、`recoverable` | 任务失败 |
| `probe.cooling` | `reason`、`recoverable`、`stage`、`ip` | 暂停流程会发出该事件，前端将其展示为可继续的等待状态 |
| `pipeline.node_started` | `pipeline_profile_name`、`region`、`domain`、`node_id`、`node_name`、`node_type`、`action` | 工作流目标进入具体节点，前端会显示为“目标 / 节点开始执行” |
| `pipeline.node_completed` | `pipeline_profile_name`、`region`、`domain`、`node_id`、`node_name`、`node_type`、`action`、`status`、`message`、`outcome`、`output_summary` | 工作流节点完成、跳过或失败，前端过程追踪会展示具体目标和阶段结果 |
| `pipeline.branch_taken` | `pipeline_profile_name`、`region`、`domain`、`node_id`、`node_name`、`node_type`、`action`、`branch_taken`、`result_count` | 分支节点命中具体 outcome，用于展示“有结果 / 无结果”等路径选择 |

`probe.progress.stage` 当前可能为：

| 阶段 | 说明 |
| --- | --- |
| `stage0_pool` | IP池完成，随 `probe.preprocessed` 推送 |
| `stage1_tcp` | TCP测延迟 |
| `stage2_trace` | 追踪探测 |
| `stage3_get` | 文件测速 |

## 8. 配置与数据结构

### 8.1 配置文件位置

配置目录由 Go 的 `os.UserConfigDir()` 决定，并追加 `CFST-GUI` 子目录。若设置 `CFST_GUI_PORTABLE_ROOT`，实际数据目录为 `${CFST_GUI_PORTABLE_ROOT}/data`；若可执行文件同目录存在 `portable.json`，则使用 `<exe-dir>/data`。更完整的字段说明见 [配置详解](./configuration.md)。

| 文件或目录 | 说明 |
| --- | --- |
| `storage.json` | 储存 bootstrap，位于默认配置目录；旧 `storage_dir` 只用于一次性迁移 |
| `desktop-config.json` | 桌面 GUI / WebUI 主要配置快照，位于固定应用数据目录 |
| `mobile-config.json` | Android app 私有目录中的当前配置快照，结构与桌面快照同构 |
| `config.json` | 兼容旧版配置结构 |
| `source-profiles.json` | 输入源档案，包含 `active_profile_id` 和 `items[].sources` |
| `desktop-draft.json` | 桌面草稿，结构同 `desktop-config.json`，用于恢复未正式保存的设置 |
| `cfip-log.txt` | 调试日志，开启 debug 后写入 |
| `exports/`、`imports/`、`backups/` | 导出、导入和本地备份目录 |

检测到旧版自定义 `storage_dir` 时，后端会尝试迁移 `desktop-config.json`、`config.json`、`cfip-log.txt`、地区数据、`source-profiles.json`、`exports/`、`imports/` 和 `backups/` 等已知文件或目录；旧目录不会自动删除。

策略管道补充说明：

- 策略列表独立存储在 `pipeline-profiles.json`，每个策略保存一份独立的 `config_snapshot`、`domain`、`region` 和 `dns_push_policy`。
- `scheduler.run_mode = pipeline` 时，调度器会读取 `pipeline-profiles.json` 中已启用策略并串行执行；`auto_dns_push = false` 只会临时抑制本次运行的 DNS 推送，不会改写策略本身。
- 工作流运行期间会推送 `pipeline.node_started`、`pipeline.node_completed` 和 `pipeline.branch_taken`，payload 携带目标名、地域、域名和节点名；前端用这些字段驱动工作流画布、任务看板和运行日志中的具体阶段展示。
- 当前支持的桌面 / WebUI / Android bridge 方法包括：`LoadPipelineProfiles`、`SavePipelineProfiles`、`SavePipelineProfile`、`DeletePipelineProfile`、`RunPipeline`、`StartPipeline`、`CancelPipeline`、`GetPipelineSnapshot`、`ListPipelineResults`。

| 字段 | 映射到 | 说明 |

| `concurrency.stage2` | `HeadRoutines` | 追踪并发线程，归一化后最大为 30 |

| `download_count` 或 `stage_limits.stage3` | `TestCount` / `Stage3Limit` | `download_count` 仅兼容读取；`stage_limits.stage3` 是阶段 3 文件测速候选上限 |
| `download_time_seconds` | `DownloadTimeSeconds` | 单 IP 下载测速时长，单位秒，默认 10，不设置最大值 |
| `download_warmup_seconds` | `DownloadWarmupSeconds` | 下载预热时间，单位秒，默认 5，最低 0 |
| `download_get_concurrency` | `DownloadGetConcurrency` | 单 IP 内部 Range GET worker 数，默认 4，范围 1-32 |
| `download_buffer_kb` | `DownloadBufferKB` | 下载读取缓冲，默认 256 KiB，范围 64-4096 KiB |
| `download_http_protocol` | `DownloadHTTPProtocol` | 下载协议，支持 `auto`、`h1`、`h2`、`h3` |
| `tcp_port` | `TCPPort` | 测速端口 |
| `port_policy` | 任务上下文 | 默认 `source_override_global`，输入源单一端口优先；多个源端口时回退全局端口并 warning |

### 8.4 配置归档和敏感信息

配置 JSON 导出包含 `app_version`、`schema_version`、`exported_at`、`config_snapshot`、`source_profiles` 和 `storage`。ZIP 配置归档固定包含 `cfst-gui-config.json`，用于本地下载、WebDAV 备份、Android SAF 导出和跨端还原。

导入归档前，后端会先把当前配置写入 `backups/` 作为 `pre-import` 备份；导入包缺少 `source_profiles` 时，会从导入快照中的 `sources` 生成默认输入源档案。

配置导出、ZIP 归档和 WebDAV 备份可能包含完整 Cloudflare API Token、WebDAV 密码、导出路径和输入源路径，必须只保存到可信位置。

## 9. 外部 HTTP 与第三方 API

### 9.1 主程序外部 HTTP 请求

| 用途 | 方法 | 地址 | 调用位置 |
| --- | --- | --- | --- |
| 版本检查 | `GET` | `https://api.github.com/repos/axuitomo/CFST-GUI/releases/latest`，并读取 `cfst-gui-update-manifest.json` | `internal/app/run.go`、`internal/app/update.go` |
| 更新包下载 | `GET` | GitHub Release asset 下载地址 | `internal/app/update.go` |
| 输入源 URL 读取 | `GET` | 用户配置的输入源 URL；GitHub Raw 可兜底到 `https://cdn.jsdelivr.net/gh/...` | `internal/app/desktop_sources.go` |
| 阶段 2 追踪探测 | `GET` | 默认从文件测速URL派生 `/cdn-cgi/trace`，可由 `trace_url` 覆盖 | `internal/task/head.go`、`internal/task/colo.go` |
| 下载测速 | `GET` | 默认 `https://speed.cloudflare.com/__down?bytes=10000000`，可由桌面配置覆盖，阶段 3 不再访问 `/cdn-cgi/trace` | `internal/task/download.go` |
| MICS抽样探测 | `HEAD` | `https://<target-host>/cdn-cgi/trace` 或解析后的路径 | `internal/mcis/probe/trace.go` |
| Cloudflare DNS | `GET`/`POST`/`PUT`/`DELETE` | `https://api.cloudflare.com/client/v4/zones/{zone_id}/dns_records` | `internal/app/cloudflare_dns.go`、`mobileapi/dns.go` |
| WebDAV 备份 | `HEAD`/`PUT`/`GET` | 用户配置的 WebDAV URL + `remote_path` | `internal/app/app_archive.go`、`mobileapi/archive.go` |
| COLO 字典更新 | `GET` | 默认 geofeed URL 或用户传入 `source_url` | `internal/app/desktop_colo_dictionary.go`、`mobileapi/colo_dictionary.go` |

说明：

- 桌面 Wails 和 CLI 模式不提供本地 HTTP 端口；Linux WebUI build tag 会启动 `/api/*` HTTP API。
- Wails bridge 不是 HTTP REST API，而是桌面运行时内的 JS 到 Go 调用；WebUI API 见第 5 节。
- 追踪探测、下载测速和 MICS抽样会通过自定义 Dial 目标让请求命中指定 IP。
- 后端 HTTP 出口统一走 `internal/httpclient`；默认优先尝试 HTTP/3，失败后回退到 TCP 上的 HTTP/1.1/2，非安全写操作不做跨协议重试。
- 阶段 3 测速 GET 会追加随机测速参数，并设置 `Cache-Control: no-store` / `Pragma: no-cache`；同时校验响应长度、Range 与可用的 Digest/MD5/SHA256 头。Cloudflare DNS、GitHub Release、更新包下载和签名 URL 不追加随机参数。
- Host Header、SNI、User-Agent、TLS 校验和调试抓包目标由 `internal/httpcfg` 统一处理；抓包目标仅在调试开启且 `debug_capture_enabled=true` 时覆盖拨号地址。

### 9.2 DNS API

当前产品通过 GUI/WebUI/Android 的 DNS 读取页直接调用 Cloudflare DNS 列表 API，不再维护外置示例脚本入口。DNS 读取页只执行查询；创建、更新和删除由工作流 `deliver_dns` 节点、定时任务 DNS 推送和测速后自动推送中的 Cloudflare 项触发。

| 用途 | 方法 | 地址 | 调用位置 |
| --- | --- | --- | --- |
| 查询 DNS 记录 | `GET` | `https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/dns_records` | `internal/app/cloudflare_dns.go`、`mobileapi/dns.go` |
| 创建 DNS 记录 | `POST` | `https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/dns_records` | `internal/app/cloudflare_dns.go`、`mobileapi/dns.go` |
| 更新 DNS 记录 | `PUT` | `https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/dns_records/{DNS_RECORD_ID}` | `internal/app/cloudflare_dns.go`、`mobileapi/dns.go` |
| 删除 DNS 记录 | `DELETE` | `https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/dns_records/{DNS_RECORD_ID}` | `internal/app/cloudflare_dns.go`、`mobileapi/dns.go` |

### 10.1 输入

| WebUI 请求 | `/api/app/{method}` JSON payload、SSE 订阅和受限文件路径 |

### 10.2 输出

| 输出类型 | 说明 |
| --- | --- |
| GUI 任务看板 | 展示任务状态、进度、警告、过程追踪和导出历史 |
| 当前结果页 | 展示本次测速结果、排序过滤、单条重测和导出位置 |
| Wails/SSE/Capacitor 事件 | `desktop:probe` 推送预处理、进度、导出、完成、失败和暂停等待事件 |
| CSV 文件 | 默认 `result.csv`，可通过桌面配置修改 |
| 配置文件和归档 | `desktop-config.json`、`source-profiles.json`、JSON 导出和 ZIP 归档 |
| 调试日志 | 开启调试时写入 `cfip-log.txt`，默认 JSONL，可选自由格式文本 |
| Cloudflare DNS | DNS 页可读取线上记录；工作流、定时任务和测速后自动推送可覆盖推送匹配记录 |
| 发行版安装包 | 输出 Windows、macOS、Linux、Android 资产和 `cfst-gui-update-manifest.json` 到 `build/release/` |

结构化调试日志每行是一条 JSON 事件，固定包含 `ts`、`level`、`event`，并在探测任务中携带 `task_id` 和阶段字段。自由格式日志使用 `debug_log_format` 中的 `{field}` 占位符从同一份脱敏后的顶层事件字段渲染文本行，默认模板为 `{ts} [{level}] {event} task={task_id} stage={stage} {message}`。`debug_log_verbosity=detailed` 保持现有完整事件，核心事件为 `probe.start`、`stage.start`、`stage.complete`、`probe.export`、`probe.complete`、`probe.failed`，并可包含 `stage.detail` 中间细节；`debug_log_verbosity=simple` 只记录 `probe.start`、`stage.complete`、`probe.export`、`probe.complete`、`probe.failed`。

常见字段包括 `stage`、`message`、`duration_ms`、`counts`、`config`、`source`、`ip`、`colo`、`tcp`、`head`、`get`、`reason`、`error`。日志会脱敏 API Token、Authorization、Cookie、密码、Secret 以及 URL query 中疑似 token/signature/auth 的参数。

当前产品不会生成外置示例脚本文件；导出结果、配置归档和调试日志均通过 GUI/WebUI/Android 流程管理。

## 11. 已知限制与注意事项

| 项目 | 当前状态 |
| --- | --- |
| REST API | 桌面 Wails/CLI 不开放 HTTP 端口；Linux WebUI 提供 `/api/health`、`/api/app/{method}`、`/api/events/probe`、`/api/files/*` |
| DNS 推送 | DNS 读取页不修改线上记录；工作流、定时任务和测速后自动推送会真实调用 Cloudflare API 创建、更新和删除匹配记录 |
| 暂停/恢复 | `CancelProbe` 会设置暂停状态并通过 `probe.cooling` 提示等待；`ResumeProbe` 唤醒当前任务，实际暂停点取决于底层阶段 hook |
| 冷却/重试 | 配置结构已预留，阶段 3 会在单 IP 时长内自动续连短文件完成或临时断流；通用冷却调度和重试策略仍不是完整任务编排器 |
| 任务历史 | 前端任务快照和结果缓存在当前会话内，重启后不会恢复历史任务 |
| 配置敏感信息 | JSON/ZIP 导出、WebDAV 备份和本地备份可能包含完整 Cloudflare Token、WebDAV 凭据和本地路径 |
| WebUI 文件访问 | 只能访问 `/data`、`storageRoot()` 和 `CFST_WEBUI_ALLOWED_ROOTS` 指定的受限根目录 |
| Android 导出 | Android 文件选择、导入和结果保存依赖系统 SAF；在线更新需要新旧 APK 使用同一签名 |
| Wails 生成代码 | `frontend/wailsjs` 可能需要通过 `wails dev` 或 `wails build` 生成 |
| 发行版平台 | 发布 Windows amd64 EXE、Linux amd64/arm64 WebUI tar.gz、macOS amd64/arm64 app.zip、Android Release APK |
| Go 工具链 | `go.mod` 固定 `go 1.26.2`；同版本 `toolchain` 行会被 `go mod tidy` 视为冗余并移除 |
| 速度单位 | 后端 `downloadSpeedMb` 按 MB/s 计算；前端文案展示为 MB/s，旧字段名 `download_mbps` 仅作兼容保留 |
| 追踪延迟 | 阶段 2 GET `/cdn-cgi/trace` 的耗时会进入 `traceDelayMs`，当前结果页展示，CSV 保持旧列格式 |

## 13. 结论

CFST-GUI 当前的核心产品形态是 Wails 桌面 GUI、Linux WebUI 和 Android 移动端桥接。对外用户接口主要是图形界面；程序级接口主要是 Wails/Capacitor 绑定方法、WebUI `/api/*`、前端 bridge 函数和 `desktop:probe` 事件。DNS 读取页已接入 Cloudflare 官方 API 且不修改线上记录；Cloudflare 推送由工作流、定时任务或测速后自动推送链路触发。
