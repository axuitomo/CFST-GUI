# CFST-GUI 功能与相关接口文档

## 1. 文档说明

本文档基于当前仓库源码整理项目功能与接口边界。

当前仓库主体并不是 Web GUI 或 REST 服务，而是一个 Go 命令行测速工具，核心能力是对指定 IP 段执行延迟测速、过滤、下载测速、结果输出，并通过脚本扩展 Hosts、dnsmasq、Cloudflare DNS、DNSPod 等场景。

也就是说：

- 仓库内没有提供浏览器页面接口
- 仓库内没有提供 HTTP 服务端 API
- 这里的“相关接口”主要指：
  - 命令行参数接口
  - Go 包内部调用接口
  - 程序运行过程中访问的外部 HTTP 接口
  - `script/` 目录内扩展脚本调用的第三方 DNS API

## 2. 项目功能总览

### 2.1 核心功能

主程序提供以下能力：

| 功能模块 | 说明 | 关键代码 |
| --- | --- | --- |
| IP 数据加载 | 从 `-ip` 参数或 `-f` 文件读取 IP/CIDR，生成待测速 IP 池 | `task/ip.go` |
| 延迟测速 | 支持 TCPing 与 HTTPing 两种模式 | `task/tcping.go`、`task/httping.go` |
| 条件过滤 | 按平均延迟上下限、丢包率上限过滤结果 | `utils/csv.go` |
| 下载测速 | 对候选 IP 逐个测速并按速度排序 | `task/download.go` |
| 地区码识别 | 从响应头提取 Cloudflare / CloudFront / Fastly / Gcore / CDN77 / Bunny 等地区码 | `task/httping.go`、`task/download.go` |
| 结果输出 | 结果打印到终端，并可导出 CSV 文件 | `utils/csv.go` |
| 版本检查 | 启动 `-v` 时访问远端检查最新版本 | `main.go` |

### 2.2 扩展功能

`script/` 目录提供了示例脚本，用于把测速结果接入其他系统：

| 脚本 | 功能说明 |
| --- | --- |
| `script/cfst_hosts.sh` | 将测速得到的最优 IP 回写到 Linux Hosts |
| `script/cfst_hosts_mac.sh` | 将测速得到的最优 IP 回写到 macOS Hosts |
| `script/cfst_hosts.bat` | 将测速得到的最优 IP 回写到 Windows Hosts |
| `script/cfst_3proxy.bat` | 用最优 IP 替换 3Proxy 配置中的旧 Cloudflare IP |
| `script/cfst_dnsmasq.sh` | 生成 `dnsmasq` 的 `cloudflare.conf` 并重启服务 |
| `script/cfst_ddns.sh` | 调用 Cloudflare API 更新 DNS 记录 |
| `script/cfst_ddns.bat` | Windows 下调用 Cloudflare API 更新 DNS 记录 |
| `script/cfst_dnspod.sh` | 调用 DNSPod API 更新 A / AAAA 记录 |

## 3. 主流程说明

主程序执行链路如下：

1. 在 `main.go` 中注册并解析命令行参数
2. 初始化随机种子 `task.InitRandSeed()`
3. 创建测速对象 `task.NewPing()`
4. 执行延迟测速 `Run()`
5. 按平均延迟过滤 `FilterDelay()`
6. 按丢包率过滤 `FilterLossRate()`
7. 执行下载测速 `task.TestDownloadSpeed(...)`
8. 导出 CSV `utils.ExportCsv(...)`
9. 打印结果 `Print()`

对应代码主链路：

```go
pingData := task.NewPing().Run().FilterDelay().FilterLossRate()
speedData := task.TestDownloadSpeed(pingData)
utils.ExportCsv(speedData)
speedData.Print()
```

## 4. 模块与内部接口

### 4.1 启动与参数入口

主入口文件：`main.go`

主要职责：

- 注册命令行参数
- 将参数写入 `task` / `utils` 包级变量
- 触发整个测速流程
- 在 `-v` 模式下检查远程版本更新

关键内部接口：

| 接口 | 说明 |
| --- | --- |
| `init()` | 参数注册、默认值处理、版本检查开关处理 |
| `main()` | 主执行入口 |
| `checkUpdate()` | 请求远程版本接口，判断是否存在新版本 |
| `endPrint()` | Windows 下等待回车，避免双击运行后窗口直接关闭 |

### 4.2 IP 数据加载模块

文件：`task/ip.go`

主要职责：

- 解析 IP/CIDR
- 支持从文本参数或文件读取
- 支持 IPv4 / IPv6
- 支持随机取样或 IPv4 全量遍历

关键内部接口：

| 接口 | 说明 |
| --- | --- |
| `InitRandSeed()` | 初始化随机种子 |
| `loadIPRanges()` | 汇总入口，读取 `-ip` 或 `-f` |
| `parseCIDR()` | 解析 IP 段 |
| `chooseIPv4()` | 生成 IPv4 测试目标 |
| `chooseIPv6()` | 生成 IPv6 测试目标 |
| `getIPRange()` | 计算 IPv4 子网可用范围 |

输入来源：

- `-ip`：直接传入逗号分隔的 IP / CIDR
- `-f`：从文件读取 IP / CIDR 列表

### 4.3 延迟测速模块

文件：`task/tcping.go`

主要职责：

- 创建测速任务
- 控制并发
- 对每个 IP 执行 TCP 或 HTTP 延迟测试
- 汇总出基础结果集

关键内部接口：

| 接口 | 说明 |
| --- | --- |
| `NewPing()` | 创建 `Ping` 实例并加载待测 IP |
| `Run()` | 启动全部延迟测速任务并排序 |
| `start()` | goroutine 包装函数 |
| `tcping()` | 执行单次 TCP 建连测时 |
| `checkConnection()` | 根据模式选择 TCPing 或 HTTPing |
| `tcpingHandler()` | 汇总单个 IP 的统计结果 |

输出结构：

- `utils.PingDelaySet`
- 元素类型为 `utils.CloudflareIPData`

### 4.4 HTTPing 模块

文件：`task/httping.go`

主要职责：

- 使用 HTTP HEAD 请求代替 TCP 建连测试
- 校验响应状态码
- 从响应头中提取地区码
- 按 `-cfcolo` 做地区过滤

关键内部接口：

| 接口 | 说明 |
| --- | --- |
| `(*Ping).httping(ip)` | 对单个 IP 做 HTTP 延迟测试 |
| `MapColoMap()` | 解析 `-cfcolo` 参数为映射表 |
| `getHeaderColo(header)` | 从响应头识别地区码 |
| `(*Ping).filterColo(colo)` | 判断地区码是否符合筛选条件 |

已适配的地区码来源：

- Cloudflare：`cf-ray`
- AWS CloudFront：`x-amz-cf-pop`
- Fastly：`x-served-by`
- Gcore：`x-id-fe`
- CDN77：`x-77-pop`
- Bunny：`server`

### 4.5 下载测速模块

文件：`task/download.go`

主要职责：

- 对延迟筛选后的候选 IP 逐个下载测速
- 使用指定 IP 作为连接目标访问测速 URL
- 统计移动平均速度
- 按下载速度排序

关键内部接口：

| 接口 | 说明 |
| --- | --- |
| `TestDownloadSpeed(ipSet)` | 下载测速总入口 |
| `checkDownloadDefault()` | 处理下载相关默认值 |
| `getDialContext(ip)` | 构造固定拨号目标，确保请求命中指定 IP |
| `downloadHandler(ip)` | 单个 IP 下载测速实现 |
| `printDownloadDebugInfo(...)` | 调试模式输出错误细节 |

测速规则摘要：

- 若 `-dd` 开启，则跳过下载测速
- 若配置了 `-sl`，程序会继续尝试直到达到 `-dn` 目标数量或队列耗尽
- 若不配置 `-sl`，默认返回全部测速结果并按速度排序

### 4.6 结果输出模块

文件：`utils/csv.go`、`utils/progress.go`、`utils/color.go`

主要职责：

- 维护结果数据结构
- 对延迟和丢包做二次过滤
- 输出 CSV
- 输出彩色表格和进度条

关键内部接口：

| 接口 | 说明 |
| --- | --- |
| `ExportCsv(data)` | 导出结果到 CSV |
| `PingDelaySet.FilterDelay()` | 按延迟范围过滤 |
| `PingDelaySet.FilterLossRate()` | 按丢包率过滤 |
| `DownloadSpeedSet.Print()` | 打印终端结果 |
| `NewBar(...)` / `Grow(...)` / `Done()` | 进度条控制 |
| `NoPrintResult()` | 是否跳过控制台打印 |

数据结构：

| 结构 | 说明 |
| --- | --- |
| `PingData` | 单 IP 的基础测速数据 |
| `CloudflareIPData` | 在 `PingData` 基础上增加丢包率缓存与下载速度 |
| `PingDelaySet` | 延迟排序结果集 |
| `DownloadSpeedSet` | 下载速度排序结果集 |

## 5. 命令行接口（CLI）

本项目最核心的对外接口是命令行参数。

### 5.1 测速控制参数

| 参数 | 含义 |
| --- | --- |
| `-n` | 延迟测速并发线程数 |
| `-t` | 单个 IP 延迟测速次数 |
| `-dn` | 下载测速目标数量 |
| `-dt` | 单个 IP 下载测速时间 |
| `-tp` | 指定测速端口 |
| `-url` | 指定 HTTPing / 下载测速地址 |

### 5.2 HTTPing 与地区筛选参数

| 参数 | 含义 |
| --- | --- |
| `-httping` | 切换为 HTTP 协议延迟测速 |
| `-httping-code` | 指定有效 HTTP 状态码 |
| `-cfcolo` | 按地区码筛选结果，仅 HTTPing 模式可用 |

### 5.3 过滤条件参数

| 参数 | 含义 |
| --- | --- |
| `-tl` | 平均延迟上限 |
| `-tll` | 平均延迟下限 |
| `-tlr` | 丢包率上限 |
| `-sl` | 下载速度下限 |

### 5.4 输入输出参数

| 参数 | 含义 |
| --- | --- |
| `-p` | 控制终端显示结果数量 |
| `-f` | 指定 IP 段文件 |
| `-ip` | 直接传入 IP 段文本 |
| `-o` | 指定结果输出文件 |

### 5.5 行为控制参数

| 参数 | 含义 |
| --- | --- |
| `-dd` | 禁用下载测速 |
| `-allip` | 测试 IPv4 网段中的全部 IP |
| `-debug` | 输出调试日志 |
| `-v` | 打印版本并检查更新 |
| `-h` | 显示帮助信息 |

## 6. 外部 HTTP 接口说明

### 6.1 主程序访问的外部接口

虽然项目没有对外暴露 Web API，但运行过程中会访问以下外部 HTTP 资源：

| 用途 | 方法 | 地址 | 调用位置 |
| --- | --- | --- | --- |
| 版本检查 | `GET` | `https://api.xiu2.xyz/ver/cloudflarespeedtest.txt` | `main.go` |
| HTTPing 测试 | `HEAD` | 默认 `https://cf.xiu2.xyz/url`，可由 `-url` 覆盖 | `task/httping.go` |
| 下载测速 | `GET` | 默认 `https://cf.xiu2.xyz/url`，可由 `-url` 覆盖 | `task/download.go` |

说明：

- 主程序本身没有本地监听端口
- 主程序本身没有 HTTP 路由或 REST endpoint
- 程序只是作为 HTTP 客户端访问远端资源

### 6.2 扩展脚本访问的第三方 API

#### Cloudflare DNS API

脚本：

- `script/cfst_ddns.sh`
- `script/cfst_ddns.bat`

使用接口：

| 方法 | 地址 | 用途 |
| --- | --- | --- |
| `PUT` | `https://api.cloudflare.com/client/v4/zones/{ZONE_ID}/dns_records/{DNS_RECORDS_ID}` | 更新 DNS 记录内容为测速得到的最优 IP |

支持两种鉴权方式：

- `X-Auth-Email + X-Auth-Key`
- `Authorization: Bearer <token>`

#### DNSPod API

脚本：`script/cfst_dnspod.sh`

使用接口：

| 方法 | 地址 | 用途 |
| --- | --- | --- |
| `POST` | `https://dnsapi.cn/Record.List` | 查询记录 ID |
| `POST` | `https://dnsapi.cn/Record.Create` | 创建 DNS 记录 |
| `POST` | `https://dnsapi.cn/Record.Modify` | 更新 DNS 记录 |

## 7. 输入与输出

### 7.1 输入

项目支持两类输入：

| 输入类型 | 说明 |
| --- | --- |
| 命令行参数 | 通过 `flag` 注入运行配置 |
| IP 列表文件 | 默认 `ip.txt`，也支持 `ipv6.txt` 或自定义文件 |

### 7.2 输出

项目支持两类输出：

| 输出类型 | 说明 |
| --- | --- |
| 终端表格输出 | 展示 IP、发送数、接收数、丢包率、平均延迟、下载速度、地区码 |
| CSV 文件输出 | 默认写入 `result.csv`，可通过 `-o` 修改或关闭 |

扩展脚本还会产生额外输出：

- `result_hosts.txt`
- `result_ddns.txt`
- `result4.csv`
- `result6.csv`
- `cloudflare.conf`
- `nowip_hosts.txt`

## 9. 结论

当前仓库的“功能 + 接口”可以归纳为以下边界：

- 核心产品形态是命令行测速工具，不是 Web GUI
- 主要对外接口是 CLI 参数
- 主要内部接口是 `task` 与 `utils` 包中的函数/结构
- 主程序只作为 HTTP 客户端访问远端测速地址和版本检查地址
- 真正涉及第三方平台 API 的，是 `script/` 目录中的 Cloudflare / DNSPod 自动更新脚本
