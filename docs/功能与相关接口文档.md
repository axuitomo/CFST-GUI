# CFST-GUI 功能与相关接口文档

## 1. 文档说明

本文档基于当前仓库源码整理 CFST-GUI 的功能、运行链路和接口边界。当前项目不是上游项目的原始形态，而是一个基于 Wails v2.12.0 + Vue 的桌面端 Cloudflare/CDN IP 测速工具，后端 Go 目标版本为 1.26.2。

当前接口边界如下：

- 桌面 GUI 默认通过 Wails 启动，前端经 `window.go.main.App` 调用 Go 后端方法。
- 仓库没有提供本地 HTTP Server、REST API 或浏览器页面路由 API。
- 文档中的“接口”主要指 Wails 后端绑定方法、前端 bridge 适配函数、Wails 事件、桌面配置结构、外部 HTTP 请求以及脚本调用的第三方 DNS API。

关键入口文件：

| 文件 | 作用 |
| --- | --- |
| `main.go` | GUI 启动入口、兼容启动判定、版本检查 |
| `gui.go` | Wails 桌面窗口启动与 `App` 绑定 |
| `app.go` | Wails 后端桥接、配置读写、测速任务执行 |
| `desktop_sources.go` | 桌面输入源读取、预览、抓取和 MICS抽样处理 |
| `desktop_probe_events.go` | `desktop:probe` 事件封装与推送 |
| `frontend/src/lib/bridge.ts` | 前端到 Wails 后端的桥接适配层 |
| `frontend/src/App.vue` | 桌面 UI 状态编排和页面调用入口 |
| `task/` | CFST TCP、追踪、文件测速核心逻辑 |
| `utils/` | CSV 导出、调试日志、进度工具 |

## 2. 功能总览

### 2.1 桌面 GUI

无参数运行时，程序启动 Wails 桌面 GUI。前端由 Vue 3 + Vite + Tailwind CSS 构建，后端由 Go 绑定 `App` 实例给 Wails。

桌面端主要页面如下：

| 页面 | 功能 |
| --- | --- |
| 任务看板 | 启动探测任务、展示进度、事件过程、警告和导出历史 |
| 当前结果 | 展示当前测速任务的结果表格、移动端结果卡片、排序过滤和导出位置 |
| 输入源 | 管理 URL、本地文件、手动输入三类 IP 来源 |
| 系统配置 | 保存 Cloudflare、导出、探测策略、高级 HTTP 和调试参数 |
| DNS 推送 | 读取 Cloudflare 线上记录，并基于测速结果或手动 IP 列表执行覆盖推送 |

### 2.2 桌面测速

桌面测速复用 CFST 核心流程，并通过 Wails 事件把任务进展同步给前端。

| 功能 | 当前行为 | 关键代码 |
| --- | --- | --- |
| 阶段 0 IP池 | 读取输入源、展开候选 IP/CIDR，并推送 `probe.preprocessed` | `desktop_sources.go`、`app.go` |
| 阶段 1 TCP测延迟 | 强制执行 TCP 建连测速，`delayMs` 定义为 TCP 平均延迟 | `task/tcping.go`、`app.go` |
| 阶段 2 追踪探测 | 对 TCP 通过候选执行 GET `/cdn-cgi/trace`，固定并发上限 6，用于筛选和地区码 | `task/head.go`、`task/colo.go` |
| 阶段 3 文件测速 | `full` 策略追加文件下载测速，可按配置使用受限并发；下载时间按单 IP 生效 | `task/download.go`、`app.go` |
| 阈值过滤 | 支持 TCP 延迟上限/下限、丢包率上限（硬上限 15%）、下载速度下限 | `utils/csv.go`、`app.go` |
| 地区码识别 | 追踪响应 body `colo=XXX` 优先，`CF-RAY` 正则备用，并兼容 CloudFront、Fastly、Gcore、CDN77、Bunny 响应头 | `task/colo.go` |
| CSV 导出 | 默认导出 `result.csv`，可配置目标目录和文件名 | `utils/csv.go`、`app.go` |
| 调试抓包 | 支持自定义 User-Agent、Host Header、SNI、调试拨号目标 | `internal/httpcfg/profile.go`、`task/request_profile.go` |

### 2.3 输入源管理

输入源随桌面配置保存，每个来源可独立设置启用状态、类型、IP 上限和 IP 处理模式。

| 输入源类型 | 字段 | 读取方式 |
| --- | --- | --- |
| `url` | `url` | 使用 HTTP GET 读取远程 IP 列表 |
| `file` | `path` | 使用本地文件路径读取 IP 列表 |
| `inline` | `content` | 使用文本框中的手动输入内容 |

候选 IP 处理模式：

| 模式 | 行为 |
| --- | --- |
| `traverse` | 解析 IP/CIDR，按顺序展开 CIDR，并按 `ip_limit` 截断 |
| `mcis` | 界面显示为 MICS抽样，将 IP/CIDR 转换为 CIDR 输入，先通过内置抽样搜索引擎和可选 IATA COLO 白名单筛选候选，再交给 CFST 做最终测速 |

### 2.4 探测策略

桌面端配置中有两个主要策略：

| 策略 | 说明 |
| --- | --- |
| `fast` | 默认策略，执行阶段 0/1/2：IP池、TCP测延迟、追踪探测，跳过文件测速 |
| `full` | 执行阶段 0/1/2/3：在追踪通过后追加文件测速 |

阶段 1 TCP 默认发包 4 次，启用跳过首包时首包不计入平均延迟和丢包统计。当前结果页、前端 `tcp_latency_ms` 和 CSV 延迟列只展示 TCP 平均延迟；阶段 2 追踪延迟通过 `traceDelayMs` / `trace_latency_ms` 提供给 UI，CSV 保持旧列格式。

阶段 2 追踪探测使用独立并发闸门，默认并发为 6，最大并发为 20。旧配置中 `concurrency.stage2` 超过 20 时会归一化为 20。阶段 3 文件测速只在 `full` 策略运行，文件测速并发固定为 1；所有追踪通过的 IP 都会串行进入文件测速，下载测速时间表示每个 IP 的单次测速时长。

高级参数包括 TCP并发线程、追踪候选上限、单 IP 下载测速时间、测速次数、端口、文件测速URL、追踪 URL、User-Agent、Host Header、SNI、追踪有效状态码、地区码过滤、调试抓包目标等。`download_count` 和 `stage_limits.stage3` 保留为兼容字段，但不再限制文件测速数量。

### 2.5 DNS 工作台

系统配置中已经预留 Cloudflare API Token、Zone ID、记录名、TTL、代理开关和备注字段；旧配置中的记录类型字段继续兼容读取。

当前实现行为：

- 桌面和移动端都会通过后端调用 Cloudflare API，要求配置完整 API Token、Zone ID 和记录名。
- `listDnsRecords()` 读取当前 Zone 下匹配记录名的 A/AAAA 线上记录，并返回 `DnsRecordSnapshot`。
- `pushDnsRecords()` 会将输入 IP 归一化并按地址族分组，IPv4 覆盖同步 A 记录，IPv6 覆盖同步 AAAA 记录。
- TTL 仅支持 60、300、600 秒三档，默认 300 秒；旧配置中的自动 TTL `1` 会归一化为 300 秒。
- 该操作不是 dry run；推送前应确认记录名、TTL、代理开关和备注配置，避免覆盖生产记录。

## 3. 运行链路

### 3.1 GUI 启动链路

```text
main()
  -> runGUI()
  -> wails.Run(...)
  -> Bind: App
  -> frontend 调用 window.go.main.App.*
```

关键点：

- `main.go` 中没有参数时默认进入 `runGUI()`。
- `gui.go` 将 `frontend/dist` 作为嵌入资源，并把 `App` 绑定给 Wails。
- 前端通过 `frontend/src/lib/bridge.ts` 封装 Wails 调用和事件监听。

### 3.2 桌面探测链路

```text
App.vue launchProbe()
  -> bridge.startProbe(payload)
  -> App.RunDesktopProbe(payload)
  -> prepareDesktopSources()
  -> emitter: probe.preprocessed(stage0_pool)
  -> App.runProbe()
  -> stage1_tcp: task.NewPing().Run().FilterDelay().FilterLossRate()，丢包率必须 <=15%
  -> stage2_trace: task.TestTraceAvailability()
  -> stage3_get: task.TestDownloadSpeed(...) 仅 full 策略运行
  -> utils.ExportCsv(...)
  -> emitter: probe.partial_export
  -> emitter: probe.completed
  -> bridge 缓存 taskSnapshots / taskResults
  -> App.vue 刷新看板、当前结果页和 DNS 推送输入
```

### 3.3 输入源预览和抓取链路

```text
App.vue inspectSource()
  -> previewDesktopSource() 或 fetchDesktopSource()
  -> App.PreviewDesktopSource() 或 App.FetchDesktopSource()
  -> processDesktopSource()
  -> loadDesktopSourceContent()
  -> buildDesktopSourceEntriesWithConfig()
  -> 可选 MICS抽样搜索
  -> 返回 preview_entries、source_status、summary
```

预览和抓取的主要区别：

| 操作 | 是否持久化状态 | 典型用途 |
| --- | --- | --- |
| 预览 | 默认不持久化，除非 `persist_state` 为真 | 检查来源是否可读、候选是否正确 |
| 抓取 | 持久化 `last_fetched_at`、`last_fetched_count`、`status_text` | 更新输入源读取状态 |

## 4. Wails 后端接口

Wails 后端绑定对象是 `App`，前端可通过 `window.go.main.App` 调用公开方法。

### 4.1 统一返回结构

桌面端主要使用 `DesktopCommandResult`：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | `string` | 业务状态码 |
| `data` | `any` | 返回数据 |
| `message` | `string` | 用户可读消息 |
| `ok` | `boolean` | 是否成功 |
| `schema_version` | `string` | Go 后端当前为 `cfst-gui-wails-v1` |
| `task_id` | `string \| null` | 关联任务 ID |
| `warnings` | `string[]` | 非致命警告 |

前端 bridge 会把返回结果归一化为自己的 `CommandResult<T>`，前端默认 schema fallback 是 `phase1-bridge-v1`。

### 4.2 当前前端直接调用的方法

| Wails 方法 | 入参 | 返回 | 说明 |
| --- | --- | --- | --- |
| `LoadDesktopConfig()` | 无 | `DesktopCommandResult` | 读取 `desktop-config.json`，不存在时返回默认配置 |
| `SaveDesktopConfig(payload)` | `{ config_snapshot }` | `DesktopCommandResult` | 保存桌面配置快照 |
| `PreviewDesktopSource(payload)` | `DesktopSourcePreviewPayload` | `DesktopCommandResult` | 读取并解析单个输入源，返回预览结果 |
| `FetchDesktopSource(payload)` | `DesktopSourcePreviewPayload` | `DesktopCommandResult` | 读取并解析单个输入源，并持久化读取状态 |
| `RunDesktopProbe(payload)` | `DesktopProbePayload` | `ProbeRunResult` | 运行桌面探测任务，期间推送 Wails 事件 |
| `SelectPath(payload)` | `{ mode, current_path, default_file_name }` | `DesktopCommandResult` | 调用系统选择器选择输入文件、导出目录或读取配置文件 |
| `OpenPath(targetPath)` | `string` | `void` 或错误 | 调用系统默认方式打开导出路径 |

### 4.3 后端兼容或辅助方法

以下方法也绑定在 `App` 上，但当前 `frontend/src/lib/bridge.ts` 没有直接使用它们：

| Wails 方法 | 返回或用途 |
| --- | --- |
| `GetHealth()` | 返回服务名、版本、配置路径、schema、Wails transport |
| `GetDefaultConfig()` | 返回旧版 `ProbeConfig` 默认值 |
| `GetStrategyPresets()` | 返回极速模式和完整模式两个预设 |
| `LoadConfig()` | 读取兼容配置 `config.json` |
| `SaveConfig(snapshot)` | 保存兼容配置 `config.json` |
| `ValidateSources(raw)` | 解析源文本并返回有效、重复、非法统计 |
| `RunProbe(req)` | 使用旧版 `ProbeRequest` 直接运行探测 |

### 4.4 `DesktopSourcePreviewPayload`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `config` | `object` | 当前桌面配置快照 |
| `persist_state` | `boolean` | 是否持久化来源读取状态 |
| `preview_limit` | `number` | 返回预览条数上限，默认 16 |
| `source` | `DesktopSource` | 单个输入源配置 |

返回 `data` 结构：

| 字段 | 说明 |
| --- | --- |
| `preview_entries` | 预览候选 IP 列表 |
| `source_status` | 更新后的来源状态 |
| `summary.action` | `预览` 或 `抓取` |
| `summary.invalid_count` | 非法 IP/CIDR 数量 |
| `summary.mode` | `traverse` 或 `mcis` |
| `summary.name` | 输入源名称 |
| `summary.total_count` | 完整候选数量 |

### 4.5 `DesktopProbePayload`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `config` | `object` | 桌面配置快照 |
| `sources` | `DesktopSource[]` | 输入源数组 |
| `task_id` | `string` | 前端生成的任务 ID，缺失时后端生成 |

### 4.6 `ProbeRunResult`

| 字段 | 说明 |
| --- | --- |
| `config` | 本次执行归一化后的 `ProbeConfig` |
| `durationMs` | 总耗时，毫秒 |
| `outputFile` | CSV 输出路径，可能为空 |
| `results` | `ProbeRow[]` |
| `source` | 输入源解析统计 |
| `sourceStatuses` | 输入源最新状态 |
| `startedAt` | 任务开始时间 |
| `summary` | 汇总统计 |
| `warnings` | 非致命警告 |
| `schemaVersion` | Go 后端 schema |

`ProbeRow` 字段：

| 字段 | 说明 |
| --- | --- |
| `ip` | IP 地址 |
| `sended` | 发送次数 |
| `received` | 成功次数 |
| `lossRate` | 丢包率，范围 0 到 1 |
| `delayMs` | TCP 平均延迟，毫秒 |
| `traceDelayMs` | 追踪延迟，毫秒 |
| `downloadSpeedMb` | 下载速度，MB/s |
| `colo` | 地区码，缺失时前端展示为 `N/A` |

## 5. 前端 Bridge 接口

前端 bridge 文件是 `frontend/src/lib/bridge.ts`。它负责：

- 校验并归一化 Wails 返回值。
- 将 Go 结构转换为 UI 更容易消费的数据结构。
- 维护当前会话内存态 `taskSnapshots`、`taskResults`。
- 监听 `desktop:probe` Wails 事件并分发给页面。

### 5.1 统一 `CommandResult<T>`

```ts
interface CommandResult<T = Record<string, unknown> | null> {
  code: string;
  data: T | null;
  message: string;
  ok: boolean;
  schema_version: string;
  task_id: string | null;
  warnings: string[];
}
```

### 5.2 Bridge 函数

| 函数 | 后端依赖 | 说明 |
| --- | --- | --- |
| `loadConfig()` | `LoadDesktopConfig()` | 读取桌面配置 |
| `saveConfig(payload)` | `SaveDesktopConfig()` | 保存桌面配置 |
| `previewDesktopSource(payload)` | `PreviewDesktopSource()` | 预览输入源 |
| `fetchDesktopSource(payload)` | `FetchDesktopSource()` | 抓取输入源并更新状态 |
| `startProbe(payload)` | `RunDesktopProbe()` | 启动探测，完成后缓存结果和任务快照 |
| `listenToProbeEvents(handler)` | Wails `EventsOn("desktop:probe")` | 订阅探测事件 |
| `openPath(targetPath)` | `OpenPath()` | 打开导出文件或目录 |
| `getTaskSnapshot(taskId)` | 前端内存 | 读取当前会话缓存的任务快照 |
| `listTaskResults(taskId, sortBy, order, filter)` | 前端内存 | 对当前会话结果排序和过滤 |
| `listDnsRecords(payload)` | `ListCloudflareDNSRecords()` / Android `ListCloudflareDNSRecords` | 读取 Cloudflare 线上匹配记录 |
| `pushDnsRecords(payload)` | `PushCloudflareDNSRecords()` / Android `PushCloudflareDNSRecords` | 将输入 IP 覆盖推送到 Cloudflare DNS |
| `stopProbe(payload)` | 无后端中断能力 | 返回已记录暂停请求的提示 |
| `resumeProbe(payload)` | 无后端恢复能力 | 返回当前无可恢复任务的提示 |

### 5.3 结果列表字段

前端将后端 `ProbeRow` 归一化为 `ProbeResult`：

| 字段 | 说明 |
| --- | --- |
| `address` | IP 地址 |
| `colo` | 地区码 |
| `download_mbps` | 下载速度，数值单位为 MB/s（字段名保留旧拼写以兼容前端缓存） |
| `export_status` | 当前固定为 `exported` |
| `stage_status` | 当前固定为 `completed` |
| `tcp_latency_ms` | 来自后端 `delayMs`，含义为 TCP 平均延迟 |
| `last_error_code` | 当前为 `null` |

支持的排序字段：`address`、`stage`、`tcp`、`download`、`export_status`。

支持的过滤项：`all`、`exported`、`pending`、`failed`。

## 6. Wails 事件接口

后端通过 `desktop_probe_events.go` 向前端推送事件，事件名固定为 `desktop:probe`。

### 6.1 事件 Envelope

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `event` | `string` | 事件类型 |
| `payload` | `object` | 事件负载 |
| `schema_version` | `string` | Go 后端 schema |
| `seq` | `number` | 单任务内递增序号 |
| `task_id` | `string` | 任务 ID |
| `ts` | `string` | RFC3339 时间 |

### 6.2 事件类型

| 事件 | payload | 说明 |
| --- | --- | --- |
| `probe.preprocessed` | `accepted`、`filtered`、`invalid`、`source_statuses`、`stage`、`total` | 输入源预处理完成，`stage` 为 `stage0_pool` |
| `probe.progress` | `stage`、`processed`、`passed`、`failed`、`total` | TCP、追踪或文件测速阶段进度 |
| `probe.speed` | `stage`、`ip`、`current_speed_mb_s`、`average_speed_mb_s`、`bytes_read`、`elapsed_ms`、`colo` | 阶段 3 文件测速实时速率样本 |
| `probe.partial_export` | `target_path`、`written` | CSV 已写出 |
| `probe.completed` | `exported`、`failed`、`passed`、`result_count`、`target_path`、`failure_summary` | 任务完成 |
| `probe.failed` | `message`、`recoverable` | 任务失败 |
| `probe.cooling` | `reason`、`recoverable` | 前端已兼容该事件，但当前后端没有主动发出 |

`probe.progress.stage` 当前可能为：

| 阶段 | 说明 |
| --- | --- |
| `stage0_pool` | IP池完成，随 `probe.preprocessed` 推送 |
| `stage1_tcp` | TCP测延迟 |
| `stage2_trace` | 追踪探测 |
| `stage3_get` | 文件测速 |

## 7. 配置与数据结构

### 7.1 配置文件位置

配置目录由 Go 的 `os.UserConfigDir()` 决定。

| 文件 | 说明 |
| --- | --- |
| `CFST-GUI/desktop-config.json` | 桌面端配置快照，当前 GUI 主要使用 |
| `CFST-GUI/config.json` | 兼容旧版配置 |
| `CFST-GUI/cfip-log.txt` | 调试日志 |

### 7.2 桌面配置快照

`ConfigSnapshot` 由前端维护，主要包含：

| 顶层字段 | 说明 |
| --- | --- |
| `cloudflare` | Cloudflare DNS 预留配置 |
| `export` | CSV 导出配置 |
| `probe` | 探测策略和高级参数 |
| `sources` | 输入源数组 |

`cloudflare` 字段：

| 字段 | 说明 |
| --- | --- |
| `api_token` | Cloudflare API Token，当前 GUI 保存但不用于真实 DNS 写入 |
| `zone_id` | Cloudflare Zone ID |
| `record_name` | DNS 记录名 |
| `record_type` | 旧字段，兼容读取；新版推送按 IP 自动识别 A/AAAA |
| `ttl` | TTL，仅支持 60、300、600 秒，默认 300 秒 |
| `proxied` | 是否开启 Cloudflare 代理 |
| `comment` | 备注 |

`export` 字段：

| 字段 | 说明 |
| --- | --- |
| `file_name` | 导出文件名，默认 `result.csv` |
| `target_dir` | 导出目录，空值表示当前工作目录 |
| `target_uri` | Android SAF 导出目标 URI，桌面端通常为空 |
| `format` | 预留字段，当前按 CSV 输出 |
| `overwrite` | 预留字段，当前后端只根据最终路径写出 |

`probe` 字段中当前会实际映射到后端测速逻辑的配置：

| 字段 | 映射到 | 说明 |
| --- | --- | --- |
| `strategy` | `ProbeConfig.Strategy` | `fast` 或 `full` |

## 8. 外部 HTTP 与第三方 API

### 8.1 主程序外部 HTTP 请求

| 用途 | 方法 | 地址 | 调用位置 |
| --- | --- | --- | --- |
| 版本检查 | `GET` | `https://api.github.com/repos/axuitomo/CFST-GUI/releases/latest`，并读取 `cfst-gui-update-manifest.json` | `main.go`、`update.go` |
| 输入源 URL 读取 | `GET` | 用户配置的输入源 URL | `desktop_sources.go` |
| 阶段 2 追踪探测 | `GET` | 默认从文件测速URL派生 `/cdn-cgi/trace`，可由 `trace_url` 覆盖 | `task/head.go`、`task/colo.go` |
| 下载测速 | `GET` | 默认 `https://speed.cloudflare.com/__down?bytes=10000000`，可由桌面配置覆盖，阶段 3 不再访问 `/cdn-cgi/trace` | `task/download.go` |
| MICS抽样探测 | `HEAD` | `https://<target-host>/cdn-cgi/trace` 或解析后的路径 | `internal/mcis/probe/trace.go` |

说明：

- 主程序本身没有本地监听端口。
- Wails bridge 不是 HTTP REST API，而是桌面运行时内的 JS 到 Go 调用。
- 追踪探测、下载测速和 MICS抽样会通过自定义 Dial 目标让请求命中指定 IP。
- Host Header、SNI、User-Agent、TLS 校验和调试抓包目标由 `internal/httpcfg` 统一处理。

### 9.1 输入

| 输入类型 | 说明 |
| --- | --- |

### 9.2 输出

| 输出类型 | 说明 |
| --- | --- |
| GUI 任务看板 | 展示任务状态、进度、警告、过程追踪和导出历史 |
| 当前结果页 | 展示本次测速结果、排序过滤、单条重测和导出位置 |
| Wails 事件 | `desktop:probe` 推送预处理、进度、导出、完成和失败事件 |
| CSV 文件 | 默认 `result.csv`，可通过桌面配置修改 |
| 调试日志 | 开启调试时以 JSONL 写入 `cfip-log.txt` |
| Cloudflare DNS | GUI 可读取并覆盖推送匹配的线上 DNS 记录 |
| 发行版安装包 | 输出 Windows、macOS、Linux、Android 资产和 `cfst-gui-update-manifest.json` 到 `build/release/` |

调试日志每行是一条 JSON 事件，固定包含 `ts`、`level`、`event`，并在探测任务中携带 `task_id` 和阶段字段。核心事件为 `probe.start`、`stage.start`、`stage.complete`、`probe.export`、`probe.complete`、`probe.failed`；逐 IP 明细只记录失败、淘汰和文件测速结果。

常见字段包括 `stage`、`message`、`duration_ms`、`counts`、`config`、`source`、`ip`、`colo`、`tcp`、`head`、`get`、`reason`、`error`。日志会脱敏 API Token、Authorization、Cookie、密码、Secret 以及 URL query 中疑似 token/signature/auth 的参数。

扩展脚本可能产生：

- `result_hosts.txt`
- `result_ddns.txt`
- `result4.csv`
- `result6.csv`
- `cloudflare.conf`
- `nowip_hosts.txt`

## 10. 已知限制与注意事项

| 项目 | 当前状态 |
| --- | --- |
| REST API | 未提供本地 HTTP 服务或 REST endpoint |
| DNS 推送 | GUI 会真实调用 Cloudflare API 创建、更新和删除匹配记录 |
| 暂停/恢复 | 前端有 `stopProbe()`、`resumeProbe()` 占位，但后端测速任务同步执行，当前不能中断已运行任务 |
| 冷却/重试 | 配置结构已预留，当前后端没有完整调度实现 |
| 任务历史 | 前端任务快照和结果缓存在当前会话内，重启后不会恢复历史任务 |
| Wails 生成代码 | `frontend/wailsjs` 可能需要通过 `wails dev` 或 `wails build` 生成 |
| 发行版平台 | 发布 Windows amd64 EXE、Linux amd64 tar.gz、macOS amd64/arm64 app.zip、Android Release APK |
| Go 工具链 | `go.mod` 固定 `go 1.26.2`；同版本 `toolchain` 行会被 `go mod tidy` 视为冗余并移除 |
| 速度单位 | 后端 `downloadSpeedMb` 按 MB/s 计算；前端文案展示为 MB/s，旧字段名 `download_mbps` 仅作兼容保留 |
| 追踪延迟 | 阶段 2 GET `/cdn-cgi/trace` 的耗时会进入 `traceDelayMs`，当前结果页展示，CSV 保持旧列格式 |

## 12. 结论

CFST-GUI 当前的核心产品形态是 Wails 桌面 GUI 和 Android 移动端桥接。对外用户接口主要是图形界面；程序级接口主要是 Wails/Capacitor 绑定方法、前端 bridge 函数和 `desktop:probe` 事件。项目没有 REST API，DNS 面板已接入 Cloudflare API，推送会真实修改线上记录。
