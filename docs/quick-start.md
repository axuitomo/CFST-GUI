# 快速上手

本文只覆盖一次测速前最常用的四步：设置导出目录、处理 COLO 词典、设置输入源、启动测速。

## 1. 设置导出目录

1. 打开“设置”页。
2. 展开“导出设置”。
3. 在“导出目录”中点击“选择目录”，选择用于保存 CSV、测速文件和调试日志的目录。
4. 按需调整“文件名”，默认会导出为 `result.csv`。
5. 保存配置。

桌面端和 WebUI 使用普通文件目录；Android 使用系统 SAF 授权目录。清除导出目录后，应用会回到默认导出位置。

## 2. 处理 COLO 词典

1. 打开“输入源”页。
2. 找到“COLO 词典”区域。
3. 点击“更新词典”，拉取 Cloudflare GEOFEED 与辅助映射源。
4. 更新完成后点击“处理词典”，生成综合、IPv4、IPv6 三份本地 COLO 文件。
5. 展开 COLO 词典状态，确认 GEOFEED、综合、IPv4、IPv6 行数已经更新。

只有在输入源 COLO 预筛、国家/COLO 筛选或 MICS 抽样需要地区映射时，才必须先处理 COLO 词典。状态里的“未覆盖”只表示部分 GEOFEED 行暂时没有派生出 COLO，不代表测速失败。

## 3. 设置输入源

1. 打开“输入源”页。
2. 点击“添加首个来源”或新增输入源。
3. 选择输入源类型：
   - URL：填写远程文本地址。
   - 本地文件：选择本机输入源文件。
   - 手动输入：直接粘贴 IP、CIDR 或域名列表。
4. 保持输入源为“已启用”。
5. 设置“处理模式”：
   - 遍历：按顺序展开并整理来源中的候选 IP。
   - MICS 抽样：先探索候选，再交给当前测速流程做最终测速。
6. 设置 IP 上限，避免一次任务读取过多候选。
7. 如需按地区预筛，填写 COLO 筛选词，并确认已经完成 COLO 词典处理。
8. 使用“预览”或“抓取”确认输入源可以读取到候选。
9. 点击“更新并保存当前输入组”保存输入源列表。

输入源会自动跳过空行和 `#` 注释；域名会先解析为 IP 后参与测速。

## 4. 测速

1. 打开“任务看板”。
2. 确认当前输入源组正确，且至少有一个输入源处于启用状态。
3. 点击“启动任务”或“开始探测”。
4. 等待任务依次完成 IP池、TCP测延迟、追踪探测；如果设置为完整模式，还会继续执行文件测速。
5. 打开“当前结果”页查看测速结果、COLO、延迟、下载速度和实际测速端口。
6. 需要保存结果时，在“当前结果”页导出 CSV；文件会写入前面设置的导出目录。

任务运行中可以在“任务看板”暂停或继续。若当前已有任务运行或暂停，需要先完成、继续或终止后再启动新任务。
