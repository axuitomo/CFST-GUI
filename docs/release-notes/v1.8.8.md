# CFST-GUI v1.8.8

## 变更摘要

- 新增上传结果 Telegram 通知：手动上传、测速后自动上传、定时任务和策略管道自动上传会生成统一通知摘要，支持在设置页配置 Bot Token / Chat ID 并发送测试通知。
- 上传结果 payload 与定时任务状态新增 `upload_notification`，桌面、WebUI、Android 原生 bridge 和 mobileapi 共用同一通知摘要结构。
- Android 在线更新下载改为应用私有 `files/update_downloads/` 目录，通过 GitHub 镜像候选并发竞速、SHA256 校验和 FileProvider content URI 拉起安装，避免把 APK 暴露到公共 Download 目录。
- Android 更新清理会移除私有更新目录中的 APK 与 `.part` 文件，并保留一次性清理旧 DownloadManager 记录的兼容逻辑。
- 主 Release workflow 在 GitHub Release 发布成功后继续调用 Container workflow，自动发布 `ghcr.io/axuitomo/cfst-gui:1.8.8`、`ghcr.io/axuitomo/cfst-gui:v1.8.8` 和 `latest` 多架构镜像；`release-preflight` 会把 GHCR 链路缺失作为发行阻塞。
- Release、Android resubmit、Container workflow、Wails、Go runtime 和 Android 默认版本号统一提升到 `1.8.8`，Android 默认 `versionCode` 提升到 `10808`。
