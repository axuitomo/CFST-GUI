# CFST-GUI v1.8.9

## 变更摘要

- Cloudflare DNS 推送会区分“无可上传 IP”“全部目标失败”和“部分完成”：分流或组合推送全部目标失败时返回失败状态，部分目标成功时保留 `partial`，定时任务状态、上传通知和 Telegram 通知不再把这些结果改写为普通完成。
- 测速后 GitHub 自动推送改为严格消费统一上传筛选后的 GitHub 结果；筛选后没有可导出行时会跳过并提示，不再回退导出完整测速结果。
- 测速后自动推送在统一上传筛选失败时会停止后续 provider，并返回明确 warning，避免 Cloudflare / GitHub 使用不一致的结果口径。
- Release、Android resubmit、Container workflow、Wails、Go runtime 和 Android 默认版本号统一提升到 `1.8.9`，Android 默认 `versionCode` 提升到 `10809`。
- 主 Release workflow 继续在 GitHub Release 发布成功后调用 Container workflow，自动发布 `ghcr.io/axuitomo/cfst-gui:1.8.9`、`ghcr.io/axuitomo/cfst-gui:v1.8.9` 和 `latest` 多架构镜像；`release-preflight` 会把 GHCR 链路缺失作为发行阻塞。
