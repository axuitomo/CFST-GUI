# CFST-GUI v1.7.3

`v1.7.3` 聚焦发行前维护体系、质量门禁和 Docker/GHCR 发布链路收口，确保桌面端、Linux WebUI、Android 与容器镜像的默认版本面统一到 `1.7.3`。

## 重点更新

- 新增完整的本地维护脚本体系，覆盖 doctor、bootstrap、lint、format、check、audit、generated artifact 校验、changed-file 检查、release preflight、bundle report、artifact inspect、secrets scan、docs check、版本提升与依赖报告。
- 新增前端 ESLint 与 Prettier 配置，并补充 GitHub Actions quality workflow，用于统一本地与 CI 的质量门禁。
- 关闭 Vite 构建默认 bundle 体积 warning；`scripts/bundle-report.sh` 默认只输出体积报告，只有显式设置阈值时才产生 warning。
- 改进生成物校验策略：默认检查重生成是否产生漂移，允许本地已同步但尚未提交的生成物通过；如需严格要求生成物相对 HEAD 干净，可设置 `CFST_VERIFY_GENERATED_STRICT=1`。
- 前端构建产物已重新同步，当前嵌入资产使用新的 hashed bundle，并移除旧 bundle。
- GitHub Release、GHCR / Docker、Android 默认版本号统一提升到 `1.7.3`，Android 默认 `versionCode` 提升到 `10703`。
- Linux WebUI 发行包继续包含 Docker Compose 上下文；GHCR workflow 将发布 `ghcr.io/axuitomo/cfst-gui:1.7.3` 与 `ghcr.io/axuitomo/cfst-gui:v1.7.3` 多架构镜像。
