# CFST-GUI v1.5

`v1.5` 以 `v1.4` 为基准，重点补齐桌面端、Linux WebUI/Docker 和 Android 三端发布链路，新增自动调度、GitHub 结果导出、CSV 编码控制、配置兼容净化和更完整的文档说明。

## 与 v1.4 对比

### 发行与平台

| 模块 | v1.4 | v1.5 |
| --- | --- | --- |
| 桌面端发布 | GitHub Release 构建 Windows、macOS、Linux WebUI 和 Android 资产 | 继续保留完整桌面端矩阵，并将默认发布版本提升到 `1.5` |
| Linux / Docker | Release 包含 Linux WebUI Docker Compose 部署包 | 增补 GHCR 镜像发布工作流，发布 `ghcr.io/axuitomo/cfst-gui:1.5` 与 `:v1.5` |
| Android 元数据 | `versionName=1.4`，默认 `versionCode=10400` | `versionName=1.5`，默认 `versionCode=10500` |
| 更新清单 | Release 生成跨平台 `cfst-gui-update-manifest.json` | 保持同一 manifest 机制，资产覆盖桌面端、Docker Compose 包和 Android APK |

### 通用能力

| 模块 | v1.4 | v1.5 |
| --- | --- | --- |
| 配置兼容 | 已统一桌面端与 Android 常用测速参数 | 读取配置、档案和归档时按当前 schema 净化，补默认值、迁移旧字段别名并忽略未知字段 |
| 配置归档 | 支持 ZIP 导入导出、本地备份/还原和 WebDAV | ZIP 归档保留 `ExportConfigArchive` / `ImportConfigArchive`，本地备份/还原入口收敛到导出/导入和 WebDAV |
| WebDAV | 支持测试、备份、还原远端配置包 | WebDAV 时间戳由后端写回，归档与档案内容写入前统一净化 |
| CSV 导出 | 基础结果导出 | 新增 CSV 编码选项和导出记录追踪，便于跨平台打开结果文件 |
| GitHub 结果导出 | 不提供结果推送到仓库的工作台 | 新增仓库写入测试和结果 CSV 推送能力，可配置 owner、repo、branch、路径模板和提交信息模板 |
| 自动调度 | 手动启动测速为主 | 新增调度状态与自动任务偏好，支持间隔/每日触发、任务运行中跳过、完成后自动 DNS 推送和 GitHub 导出 |

### 探测与网络

| 模块 | v1.4 | v1.5 |
| --- | --- | --- |
| 请求配置 | 已支持 HTTP 协议选择、状态码、预热和丢包率等字段 | 增强请求配置、探测事件和结果处理，提升桌面端、WebUI、Android 之间的参数一致性 |
| 结果视图 | 展示当前任务和结果 | 增强结果刷新、筛选、排序、单条重测和 GitHub 导出操作 |
| 调试与日志 | 支持结构化调试日志 | 扩展日志模式、捕获地址和导出状态记录，便于定位测速链路问题 |

### 桌面端 / WebUI / Android

| 模块 | v1.4 | v1.5 |
| --- | --- | --- |
| 桌面端 | Wails GUI、托盘、设置页和在线更新 | 配置保存、档案切换、导入导出和状态写回路径统一走兼容净化 |
| WebUI | Linux WebUI 提供 `/api/app/{method}`、SSE 和受限文件 API | WebUI 方法矩阵补齐 GitHub 导出、调度状态、配置归档和 WebDAV 能力说明 |
| Android | 读取 APK 真实版本，配置字段与桌面端对齐 | 移动端 `mobile-config.json` 与桌面配置同构，归档、档案和储存路径同样执行兼容净化 |
| 前端桥接 | Wails、WebUI、Capacitor 三端桥接 | 移除 `BackupConfigArchive` / `RestoreConfigArchive` 本地桥接入口，保留 ZIP 导入导出与 WebDAV 备份还原 |

## 简略更改点

- 发布版本默认值统一更新为 `1.5`，Android 默认 `versionCode` 更新为 `10500`。
- 新增 GHCR 容器镜像发布，Docker Compose 包继续随 Linux WebUI Release 资产提供。
- 新增自动调度、GitHub 结果导出、CSV 编码和导出记录能力。
- 配置读取和写入路径统一做 schema 兼容净化，降低旧配置升级风险。
- 配置归档 API 收敛为导出/导入 ZIP，本地备份/还原入口改由显式导出/导入或 WebDAV 承担。
- 文档补充三端 API、配置字段、环境变量、Android 和 Docker 发布说明。
