# CFST-GUI v1.6

`v1.6` 以 `v1.5` 为基准，重点收敛统一上传筛选、输入源档案与共享核心，增强更新下载链路，并把桌面端、Linux WebUI/Docker Compose 与 Android 的公开版本面统一提升到 `1.6`。

## 与 v1.5 对比

### 发行与平台

| 模块 | v1.5 | v1.6 |
| --- | --- | --- |
| GitHub Release | 发布 Windows、Linux WebUI、macOS amd64、macOS arm64 和 Android 资产 | 继续保留完整桌面矩阵 + Android Release APK，Release 正文与更新清单统一切到 `1.6` |
| Android 元数据 | `versionName=1.5`，默认 `versionCode=10500` | `versionName=1.6`，默认 `versionCode=10600` |
| 更新清单 | 生成跨平台 `cfst-gui-update-manifest.json` | 保持同一 manifest 结构，发布版本、下载地址和桌面/Android 更新识别统一对齐 `1.6` |

### 上传工作流 / Cloudflare / GitHub

| 模块 | v1.5 | v1.6 |
| --- | --- | --- |
| 上传入口 | Cloudflare 自动推送、GitHub 结果导出和手动操作各自消费结果 | 新增统一上传选择器，定时任务、GitHub 导出和 DNS 页面“从当前结果推送”共用同一筛选口径 |
| 共享上传筛选 | 没有独立的上传规则层 | 新增 `upload.shared_filter.enabled`、`status`、`ip_version`、`colo_allow`、`colo_deny`、`max_tcp_latency_ms`、`max_trace_latency_ms`、`min_download_mbps`、`max_loss_rate` |
| 目标数量控制 | 上传数量主要跟随现有结果流 | 新增 `upload.cloudflare.top_n` 与 `upload.github.top_n`，Cloudflare 和 GitHub 可分别限制上传 Top N |
| Cloudflare / GitHub 一致性 | 手动与定时流程可能出现筛选差异 | 统一复用同一筛选结果，空结果时会明确提示 warning，减少“界面结果”和“上传结果”不一致 |

### 输入源与档案

| 模块 | v1.5 | v1.6 |
| --- | --- | --- |
| 输入源档案 | 支持保存、切换和基本兼容净化 | 最近运行输入源档案可自动更新或补建，当前输入源保存/切换/删除后的状态回写更完整 |
| 输入源加载 | 桌面端与 Android 仍保留较多重复实现 | URL 归一化、远程读取、候选构建和 MCIS 公共流程收敛到共享核心，桌面端与移动端行为更一致 |
| 远程输入源 | 远程 URL 处理逻辑分散 | 桌面端继续保留 raw GitHub -> jsDelivr 兜底与 warning，移动端继续保持单次直连策略，跨端差异改为显式注入 |

### 更新与下载

| 模块 | v1.5 | v1.6 |
| --- | --- | --- |
| 更新信息解析 | 基础 Release / manifest 解析 | 更新信息选择、版本展示和前端状态同步更稳定，桌面端更新页可直接复用统一 app info |
| 下载链路 | 以 GitHub Release 原始下载地址为主 | GitHub Release 资产下载地址统一改写为 `xget` 镜像直链，降低更新下载失败概率 |
| 安装反馈 | 已有桌面端在线更新入口 | 更新检查、下载和安装结果反馈补齐，用户更容易判断当前是否已获取可安装的新版本 |

### 兼容与内核

| 模块 | v1.5 | v1.6 |
| --- | --- | --- |
| 公开配置 | 已兼容常用探测、归档和导出字段 | `upload.*` 正式进入公开配置文档，旧快照、档案和归档读取时继续补默认值并兼容旧字段别名 |
| 跨端共享核心 | 桌面端、WebUI、Android 仍有多处重复 source runtime | 新增 `internal/appcore` 共享核心，输入源处理、Cloudflare、档案与部分探测流程继续收敛 |
| 结果与解析一致性 | 基础结果处理可用 | 端口、结果、source 解析与共享运行时继续增强，减少桌面端与 Android 的边缘行为漂移 |

## 简略更改点

- 新增统一上传筛选与每目标 Top N，Cloudflare / GitHub 可以按不同数量和规则输出结果。
- 公开 `upload.shared_filter.*`、`upload.cloudflare.top_n`、`upload.github.top_n` 配置，并同步到文档。
- 输入源档案和最近运行输入源保存逻辑增强，桌面端与 Android 的输入源处理继续共享化。
- 桌面端在线更新与下载链路增强，GitHub Release 资产下载改走 `xget` 镜像直链。
- 发布默认版本统一更新为 `1.6`，Android 默认 `versionCode` 更新为 `10600`。
- Linux WebUI Docker Compose 包继续随主 Release 提供；GHCR 工作流默认版本同步到 `1.6`，但不纳入本次主 Release 资产链路。
