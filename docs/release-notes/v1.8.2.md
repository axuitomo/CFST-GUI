# CFST-GUI v1.8.2

## 变更摘要

- 修复 Android WebView 中输入框被软键盘遮挡的问题：Activity 使用 `adjustResize`，前端按 `visualViewport` 同步可视高度，并在键盘打开时隐藏底部导航。
- 修复 Android 结果页筛选下拉显示为系统白色大面板的问题：移动端结果筛选改为应用内 picker，支持点外关闭、滚动关闭、Esc 关闭和基础 ARIA 状态。
- Android 原生层从 Java plugin 迁移为 Kotlin plugin，`CfstPlugin.kt` 保持 Capacitor 入口，SAF、导入导出、存储迁移、在线更新、通知权限、前台服务和调度能力拆分到 `Android*` Kotlin 文件，并配套 Kotlin 单元测试。
- Android 发布基线同步到 Capacitor `8.4.0`、Cordova Android `15.0.0`、AGP `9.2.1`、Gradle `9.5.1`、AGP 9 内置 Kotlin（顶层 KGP classpath `2.4.0`）、SDK platform `android-37.0`、Build Tools `37.0.0`、cmdline-tools `20.0` 和 NDK `29.0.14206865`。
- Android 构建要求 JDK 24（当前验证环境为 `24.0.2`），并把 Android 子项目 compile options 统一覆盖为 Java 24 bytecode；发布预检会校验 Kotlin `2.4.0`、SDK 37、Cordova Android 15、Capacitor 8.4.0 lockfile 和运行时默认版本。
- AndroidX Core 升到 `1.19.0`，compile SDK 对应升到 `android-37.0`。
- Android `FileProvider` 安装更新 APK 时只暴露 app 私有 `files/updates/`，并由 `android-doctor` 检查源码侧通知权限、前台服务类型、WorkManager 调度、更新清理 receiver 和 FileProvider 边界；`check-android.sh` 会继续检查 APK 产物内最终 manifest 与敏感组件导出状态。
- 前端工具链同步到 Vite 8、Tailwind CSS 4、TypeScript 6、`@vitejs/plugin-vue` 6 和 `vue-tsc` 3；Tailwind 改由 `@tailwindcss/vite` 接入，PostCSS 配置只保留 Autoprefixer，并刷新 `frontend/dist` hashed assets。
- GitHub Release、Android resubmit、GHCR / Docker、Wails 和 Android 默认版本号统一提升到 `1.8.2`，Android 默认 `versionCode` 提升到 `10802`。
- 新增 Android splash 多密度 PNG 资源，保持移动端启动资源与发行基线同步。
