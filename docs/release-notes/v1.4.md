# CFST-GUI v1.4

`v1.4` 以 `v1.3` 为基准，重点修复 Android 版本号显示异常，并补齐桌面端与安卓端共用的测速参数、设置页和发布元数据一致性。

## 与 v1.3 对比

### 通用能力

| 模块 | v1.3 | v1.4 |
| --- | --- | --- |
| 版本信息 | 桌面端与 Android 端存在不同版本来源，Android 端可能显示旧的 `1.2` | 统一发布版本为 `1.4`，Android 端读取 APK 实际 `versionName` |
| 在线更新 | 按 GitHub Releases 与 manifest 匹配资产 | 保持兼容，并确保 Android 更新检查使用真实当前版本 |
| 测速默认值 | HTTPing 状态码默认可为空；丢包率上限默认与最大值耦合 | HTTPing 默认 `0` 不按状态码筛选；丢包率默认 `0.15`、可调上限扩展到 `1.00` |
| 下载测速 | 单 IP 测速时间有最低 8 秒限制 | 新增下载预热秒数，测速时间改为正数即可 |
| 配置结构 | 未包含下载预热字段 | 桌面端与 Android 端配置快照均包含 `download_warmup_seconds` |

### 桌面端

| 模块 | v1.3 | v1.4 |
| --- | --- | --- |
| 单实例 | 可重复启动多个窗口/进程 | 新增单实例锁，第二次启动会拉起主窗口 |
| 托盘后台 | 托盘启动/退出存在重复调用风险 | 托盘启动与退出增加一次性保护，并绑定 UI 线程处理 |
| 设置页 | 多区域平铺，移动宽度下信息密度较高 | 改为折叠分组，在线更新、储存、备份、档案、测速等区域更易浏览 |
| 输入源保存 | 输入源页保存按钮容易触发档案保存语义 | 保存按钮改为直接保存当前配置，降低误操作 |
| CLI 说明 | 丢包率帮助文案仍按 `0.15` 上限描述 | 文案同步为 `0.00~1.00`，与实际校验一致 |

### Android 端

| 模块 | v1.3 | v1.4 |
| --- | --- | --- |
| 版本显示 | 设置页可能显示写死的旧版本 `1.2` | 修复为显示 APK 实际版本，本版应显示 `1.4` |
| APK 元数据 | `versionName=1.3`，`versionCode=4` | `versionName=1.4`，默认 `versionCode=10400` |
| 更新检查 | 使用写死版本判断是否有新版 | 使用安装包真实版本判断更新状态 |
| 测速参数 | 与桌面端存在下载预热、默认状态码等字段差异 | 与桌面端共享下载预热、HTTPing 默认状态码和丢包率规则 |
| 发布构建 | Android 版本需要手动同步 | Gradle 支持读取 `CFST_VERSION`，tag 发布时自动使用当前版本 |
