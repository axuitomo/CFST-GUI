# CFST-GUI v1.8.6

## 变更摘要

- 修复 Android 前台探测服务重复启动路径：重复手动启动或定时启动不再停止正在运行的前台服务，任务结束时会使用最新 `startId` 正确收尾，避免误杀当前任务或留下残留服务。
- 移动端已有运行/暂停任务时，新探测会返回当前真实 `task_id`，前端后续暂停、恢复、取消和状态读取不再误指向新请求的 task_id。
- 输入源内存缓存现在会把 loader panic 转换为读取错误并释放等待者，避免多个任务共享同一输入源读取时因异常导致等待 goroutine 永久卡死。
- 输入源读取链路补充内存去重、远程 URL 条件缓存、GitHub raw/jsDelivr fallback 诊断和生命周期审计，桌面端、移动端和定时任务的输入源状态更容易定位。
- Android 后台链路继续增强：通知栏保活、更新下载/安装、运行时状态恢复、定时任务和配置归档的移动端行为同步收口。
- Docker / GHCR 文档补充一条命令启动、Watchtower 热更新和数据持久化说明；WebUI GHCR 发布链路继续保持 `linux/amd64` 与 `linux/arm64` 多架构镜像。
- Release、Android resubmit、Container workflow、Wails、Go runtime 和 Android 默认版本号统一提升到 `1.8.6`，Android 默认 `versionCode` 提升到 `10806`。
