# CFST-GUI v1.7

`v1.7` 以 `v1.6` 为基准，重点收敛 Android 长任务稳定性、任务恢复/结果落盘链路、桌面端与移动端设置页信息架构，以及 Linux WebUI / Docker 与多架构发布面的统一。

## 与 v1.6 对比

### 发行与平台

| 模块 | v1.6 | v1.7 |
| --- | --- | --- |
| GitHub Release | 发布 Windows、Linux WebUI amd64/arm64、macOS amd64/arm64 和 Android 资产 | 继续保留完整桌面矩阵 + Android Release APK，并把默认版本面统一提升到 `1.7` |
| Android 元数据 | `versionName=1.6`，默认 `versionCode=10600` | `versionName=1.7`，默认 `versionCode=10700` |
| Docker / GHCR | 支持手动发布多架构 GHCR 镜像 | 继续保留 `linux/amd64` + `linux/arm64` 多架构镜像，并同步到 `1.7` 版本标签 |

### Android 稳定性与长任务

| 模块 | v1.6 | v1.7 |
| --- | --- | --- |
| 前台服务骨架 | 已具备基础前台服务和任务执行骨架 | 长任务执行、通知更新、快照落盘和任务恢复链路继续增强，长时运行更稳定 |
| 任务恢复 | 可恢复快照与已落盘结果 | 恢复语义更完整，区分运行中会话、暂停态与仅持久化结果，减少误判和重复恢复 |
| 文件与 SAF 储存 | 基础 bridge 可用 | Android 运行时储存桥接、配置镜像和结果缓存链路继续收敛，减少配置读取/写回抖动 |
| 16KB 页对齐 | 尚未固化到默认构建链路 | `gomobile` 产物默认启用 16KB ELF 对齐，同时保持对 4KB 设备兼容，并在构建阶段自动校验 |

### 桌面端 / Android 设置与任务体验

| 模块 | v1.6 | v1.7 |
| --- | --- | --- |
| 设置页结构 | 设置内容逐步聚合，但仍有提示卡与操作分散 | 删除顶部“系统配置”提示卡，在同一页内按通用设置、数据存储、网络任务、自动化导出和安全诊断分组组织 |
| 保存配置入口 | 设置页与输入源页按钮风格不完全统一 | 保存配置入口位置与按钮样式统一，桌面端更贴近成品稿，移动端纵向阅读也更清晰 |
| 任务看板与结果页 | 已降低闪退风险，但仍在持续收敛 | 任务恢复、结果呈现与历史/输出链路继续优化，减少重复操作和状态迷失 |

### Linux WebUI / Docker / 更新

| 模块 | v1.6 | v1.7 |
| --- | --- | --- |
| Linux 发布 | 提供 amd64/arm64 WebUI bundle、Compose 和本地运行入口 | 延续双架构 WebUI bundle、Compose 与本地运行入口，并与 `1.7` 更新清单保持一致 |
| 更新清单 | 跨平台 `cfst-gui-update-manifest.json` 可用 | 继续复用同一 manifest 结构，Linux/Android/桌面端资产识别和下载版本统一对齐 `1.7` |
| Docker 运行面 | GHCR 多架构镜像与 Linux bundle 并存 | Docker 版本标签和 Linux bundle 默认版本同步提升到 `1.7` |

## 简略更改点

- Android 长任务、任务恢复、快照/结果持久化与前台服务链路继续增强。
- Android 构建默认启用 16KB 页对齐，并在脚本中自动校验 JNI 库与 APK 对齐状态。
- 设置页删除“系统配置”提示卡，改为同页分组组织，兼顾桌面端大屏扫描与安卓端纵向阅读。
- 桌面端输入源页与设置页的“保存配置”入口位置和样式统一，贴近新版设计稿。
- 默认发行版本统一更新为 `1.7`，Android 默认 `versionCode` 更新为 `10700`。
- Linux WebUI / Docker Compose / GHCR 多架构版本面与 `1.7` Release 对齐。
